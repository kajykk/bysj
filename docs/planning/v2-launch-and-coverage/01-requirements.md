## 1. 迭代目标

### 1.1 总体目标

**一次性解决所有上线阻塞问题**，达到真正的上线就绪状态：

1. **核心功能闭环**: 所有 P0 功能完整可用
2. **部署就绪**: 前端构建、后端启动、健康检查全部验证通过
3. **测试覆盖 80%**: 后端单元/集成测试、前端单元测试全面覆盖
4. **CI/CD 完整**: GitHub Actions 自动构建、测试、覆盖率检查
5. **文档完整**: 部署文档、操作手册、API 文档齐全

### 1.2 成功标准

| 标准 | 目标 | 验收方式 |
|---|---|---|
| 前端生产构建 | 构建成功 | CI 验证 |
| 后端服务启动 | 启动成功 | CI 验证 |
| 健康检查 | /health 返回 ok | CI 验证 |
| 核心 API | 所有 P0 API 测试通过 | 自动化测试 |
| 后端覆盖率 | >= 80% | pytest-cov |

| E2E 测试 | 核心流程通过 | Playwright |

## 5. 风险与缓解

| 风险 | 影响 | 缓解措施 |
|---|---|---|
| Windows 本地测试受限 | 无法本地验证 | 完全依赖 CI/Docker 验证 |
| GitHub 推送失败 | 无法触发 CI | 使用 GitHub API 或手动上传 |
| 测试数据复杂 | 测试效率低 | 使用工厂模式 +  fixtures |
| 第三方依赖 | 测试不稳定 | Mock/Stub 替代 |
| 时间紧迫 | 质量风险 | 优先 P0，P1 可后置 |
