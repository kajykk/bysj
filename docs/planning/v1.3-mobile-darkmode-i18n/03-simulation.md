# Round 1 Simulation - 场景推演

> **迭代**: v1.3-mobile-darkmode-i18n
> **时间**: 2026-04-27

---

## 场景 1: 移动端首次访问

### 用户故事
用户使用手机浏览器首次访问系统。

### 推演步骤
1. 用户打开系统 URL
2. 系统检测为移动端（屏幕宽度 < 768px）
3. 加载移动端布局（底部导航 + 主内容区）
4. 检测系统主题为深色
5. 自动应用深色主题

### 预期结果
- ✅ 布局适配正常，无横向滚动
- ✅ 底部导航显示正确
- ✅ 深色主题自动应用

### 风险评估
- **低**: Element Plus 组件在移动端表现良好
- **中**: 图表在移动端可能需要缩放

---

## 场景 2: 主题切换

### 用户故事
用户在设置页面切换主题。

### 推演步骤
1. 用户进入设置页面
2. 点击主题切换按钮（light -> dark）
3. 系统应用 `html.dark` 类名
4. Element Plus 组件自动适配
5. ECharts 图表切换为 dark 主题
6. 用户偏好保存到 localStorage

### 预期结果
- ✅ 主题切换无闪烁
- ✅ 所有组件颜色正确
- ✅ 图表颜色适配
- ✅ 刷新后保持主题

### 风险评估
- **低**: CSS 变量切换性能良好
- **低**: localStorage 持久化可靠

---

## 场景 3: 语言切换

### 用户故事
用户从中文切换到英文。

### 推演步骤
1. 用户点击语言选择器
2. 选择 "English"
3. 系统调用 `loadLocaleMessages('en-US')`
4. 动态加载英文语言包
5. 界面文本即时更新
6. 日期格式切换为 MM/DD/YYYY

### 预期结果
- ✅ 语言切换无卡顿
- ✅ 所有文本正确翻译
- ✅ 日期格式本地化
- ✅ 刷新后保持语言

### 风险评估
- **低**: 按需加载性能良好
- **中**: 需要确保所有文本都有翻译键

---

## 场景 4: 深色模式下的图表显示

### 用户故事
用户在深色模式下查看风险趋势图表。

### 推演步骤
1. 用户进入仪表盘
2. 系统检测到深色模式
3. ECharts 使用 dark 主题配置
4. 图表背景、坐标轴、文字颜色适配

### 预期结果
- ✅ 图表背景为深色
- ✅ 坐标轴和文字为浅色
- ✅ 数据系列颜色对比度足够

### 风险评估
- **中**: 需要为 ECharts 配置 dark 主题
- **低**: 颜色对比度符合 WCAG 标准

---

## 场景 5: 移动端表单提交

### 用户故事
用户在手机上完成风险评估表单。

### 推演步骤
1. 用户点击"开始评估"
2. 表单页面加载
3. 输入框自动聚焦
4. 虚拟键盘弹出
5. 用户填写并提交
6. 显示提交成功提示

### 预期结果
- ✅ 表单布局适配移动端
- ✅ 输入框聚焦正常
- ✅ 提交按钮可点击

### 风险评估
- **低**: Element Plus 表单组件支持移动端
- **中**: 长表单可能需要分页
