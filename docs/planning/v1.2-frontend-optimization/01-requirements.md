# 项目需求文档 (PRD) - v1.2 前端体验优化

## 1. 项目概述

### 1.1 背景
v1.1 迭代已完成生理信号抑郁预测模型的训练与集成，后端推理能力已就绪。当前前端（Vue 3 + Element Plus + Vite）在用户体验、视觉一致性、交互流畅度方面存在优化空间，亟需通过一次专门的前端优化迭代提升产品质感与可用性。

### 1.2 目标
对现有前端进行全面体验优化，提升页面加载性能、交互响应速度与视觉一致性，使系统达到毕业答辩展示水准。

### 1.3 目标用户
- **学生用户**：需要直观、快速地查看抑郁风险报告与趋势
- **咨询师**：需要高效处理预警、管理用户
- **管理员**：需要清晰的数据概览与系统配置

## 2. 详细功能设计

### 2.1 模块 A: 全局样式与布局优化

#### 2.1.1 页面: 主布局 (MainLayout)
**路径**: `/` (所有子页面的父级布局)

**UI 元素清单**:
| 元素名称 | 类型 | 验证规则 | 默认值 | 交互逻辑 | 异常处理 | 权限 |
|---|---|---|---|---|---|---|
| 侧边栏导航 | Menu | - | 根据角色渲染 | 点击切换路由，当前项高亮 | - | 按角色 |
| 顶部 Header | Header | - | - | 显示用户名、角色标签、通知铃铛 | 通知加载失败静默处理 | 登录后 |
| 面包屑导航 | Breadcrumb | - | 自动解析路由 | 点击可返回上级 | - | - |
| 页面内容区 | Main | - | - | 路由切换时带过渡动画 | - | - |

**优化点**:
1. 侧边栏增加折叠/展开功能（图标模式）
2. 顶部 Header 增加全局搜索入口（快速跳转页面）
3. 页面切换增加 `fade-slide` 过渡动画
4. 统一所有页面的 padding/margin 规范（24px 标准间距）

#### 2.1.2 页面: 登录页 (LoginPage)
**路径**: `/login`

**UI 元素清单**:
| 元素名称 | 类型 | 验证规则 | 默认值 | 交互逻辑 | 异常处理 | 权限 |
|---|---|---|---|---|---|---|
| 用户名 | Input | Required | - | 回车聚焦密码 | 显示错误提示 | 公开 |
| 密码 | Input | Required | - | 支持显隐切换 | 显示错误提示 | 公开 |
| 登录按钮 | Button | - | - | Loading 状态 | 弹出错误 Toast | 公开 |

**优化点**:
1. 增加登录页背景动效（粒子效果或渐变动画）
2. 表单增加 enter 键提交支持
3. 错误提示使用 Element Plus Message 而非 alert
4. 增加"记住我"功能（localStorage 保存用户名）

### 2.2 模块 B: 用户端体验优化

#### 2.2.1 页面: 用户仪表盘 (UserDashboard)
**路径**: `/user/dashboard`

**优化点**:
1. 统计卡片增加数字滚动动画（count-up 效果）
2. 风险趋势图表增加时间范围切换（7天/30天/90天）
3. 最近测评卡片增加空状态插图（Empty State Illustration）
4. 未读预警列表增加标记已读功能

#### 2.2.2 页面: 风险评估 (UserRiskPage)
**路径**: `/user/risk`

**优化点**:
1. **风险报告标签页**:
   - 仪表盘进度条增加颜色渐变动画
   - 风险因子表格增加排序功能
   - 建议列表使用卡片式布局替代纯文本
   - 趋势图增加数据点 hover 提示
   - **报告同步机制**: 结构化评估/文本分析/生理数据提交后，自动调用 `loadReport()` 刷新风险报告数据，若刷新失败则显示 Toast 提示但保留用户当前标签页

2. **结构化评估标签页**:
   - 表单增加分步向导（Stepper）模式，降低认知负荷
   - 滑块组件增加实时数值显示
   - 提交后增加结果预览弹窗（Modal）
   - **表单草稿自动保存**: 每 30 秒自动保存到 localStorage（键: `dws_structured_draft`），页面刷新后自动恢复；提交成功后自动清除草稿缓存

3. **文本分析标签页**:
   - 文本输入框增加字符计数器
   - 情绪标签使用彩色 Tag 组件
   - 分析结果使用动画过渡展示

4. **多模态融合标签页**:
   - JSON 输入框增加格式化/验证提示
   - 结果展示使用卡片式布局

5. **实验评估标签页**:
   - 训练面板增加进度条可视化
   - 图表增加加载状态（Skeleton）
   - 日志查看器增加搜索/过滤功能

6. **生理数据标签页**:
   - 表单增加数据合理性提示（如心率范围 60-100）
   - 历史数据表格增加趋势箭头（↑↓）

## 3. 非功能需求

### 3.1 性能
- 页面首次加载时间 < 2s（Lighthouse Performance Score >= 80）
- 路由切换时间 < 300ms
- 图表渲染时间 < 500ms
- **测量标准**: 使用 Lighthouse CLI 在本地生产构建上测量，网络环境为模拟 Fast 3G

### 3.2 视觉一致性
- 统一使用 Element Plus 设计规范
- 所有页面遵循 8px 网格系统
- 颜色使用 CSS 变量管理，支持主题切换基础
- **响应式断点**: 
  - 桌面端: >= 1280px (默认布局)
  - 平板端: 768px - 1279px (侧边栏自动折叠)
  - 移动端: < 768px (简化布局，隐藏非核心元素)

### 3.3 交互体验
- 所有按钮点击有反馈（Loading / Ripple）
- 表单提交有防重复提交机制
- 所有异步操作有加载状态提示
- **表单草稿自动保存**: 结构化评估表单在填写过程中每 30 秒自动保存到 localStorage，刷新页面后可恢复

### 3.4 可访问性
- 支持键盘导航（Tab 顺序合理）
- 图片和图标有 alt 文本
- 颜色对比度符合 WCAG 2.1 AA 标准
- **验证工具**: 使用 axe-core 进行自动化可访问性测试

### 3.5 浏览器兼容性
- Chrome >= 90
- Edge >= 90
- Firefox >= 88
- Safari >= 14
- **不支持 IE11**

## 4. 假设与约束
- 假设后端 API 在 v1.2 迭代中保持稳定，不引入破坏性变更
- 约束：优化范围仅限于前端，不涉及后端逻辑修改
- 约束：需保持与现有 Element Plus 组件库的兼容性
- **localStorage 键名规范**: 所有 localStorage 键必须使用前缀 `dws_`（Depression Warning System），避免与其他应用冲突。例如：`dws_username`, `dws_prediction_history_v1`
