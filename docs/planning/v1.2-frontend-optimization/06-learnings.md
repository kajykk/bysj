# 经验总结 (Learnings) - v1.2 前端体验优化

> 本文件用于记录迭代过程中的经验、教训与最佳实践。
> 在每一轮迭代结束后更新。

---

## Round 1 经验

### 发现的问题
1. **响应式设计缺失**: 初始 PRD 未定义响应式断点，可能导致移动端适配混乱
2. **预警处理闭环不完整**: Drawer 组件内的操作按钮和状态流转未定义，导致业务流程断裂
3. **导出加载状态未定义**: 大数据量导出时的用户体验未考虑
4. **内存泄漏防护不足**: ECharts 实例在快速路由切换时可能泄漏

### 教训
- 前端需求不能仅关注视觉效果，必须定义完整的交互闭环
- 任何涉及异步操作的流程都必须定义 Loading 态和错误态
- 组件生命周期管理（尤其是第三方库实例）必须在需求阶段明确

## Round 2 经验

### 修复措施
1. **响应式设计**: 补充了桌面端(>=1280px)、平板端(768-1279px)、移动端(<768px)的断点定义
2. **预警处理闭环**: 在 Drawer 内定义了状态流转按钮("标记已处理"/"升级风险"/"转介")、必填备注、自动刷新机制
3. **导出加载状态**: 定义了 Loading 状态、进度显示、分页导出、错误处理
4. **内存泄漏防护**: 新增 ChartContainer 组件，封装 ECharts 的初始化、更新、销毁逻辑

### 新增最佳实践
- **localStorage 键名规范**: 使用 `dws_` 前缀避免命名冲突
- **浏览器兼容性矩阵**: 明确定义支持范围（Chrome >= 90, Edge >= 90, Firefox >= 88, Safari >= 14）
- **表单草稿自动保存**: 每 30 秒自动保存，提交后清除，提升用户体验

### 技术调研结论
- ECharts 内存泄漏防护: 使用 `dispose() + ResizeObserver + nextTick` 方案
- 表单草稿自动保存: 建议使用 VueUse 的 `useLocalStorage` + `debounceFilter`
- Element Plus 按需引入: 已配置 `unplugin-vue-components`，无需额外优化
