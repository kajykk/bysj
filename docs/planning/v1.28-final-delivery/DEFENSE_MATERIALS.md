## 1. 研究背景与意义

### 1.1 心理健康问题的全球性挑战

心理健康障碍已成为21世纪最严峻的公共卫生挑战之一。根据世界卫生组织（WHO）2024年报告：

- **全球约 9.7 亿人**（约每 8 人中 1 人）患有精神障碍，其中抑郁障碍和焦虑障碍是最常见的两种类型
- 抑郁症是全球致残的首要原因，每年导致约 **70 万人自杀身亡**
- 在 **15-29 岁**年龄段，自杀是第四大死亡原因
- COVID-19 大流行后，全球抑郁和焦虑患病率增加了 **25% 以上**

### 1.2 高校心理健康筛查的现实困境

高校学生群体是心理健康问题的高发人群，但传统筛查方式面临多重瓶颈：

| 问题层面 | 具体表现 | 导致后果 |
|:---|:---|:---|
| **覆盖率低** | 依赖年度普查，动态监测缺失 | 学生心理状态变化无法及时发现 |
| **效率瓶颈** | 问卷填写负担重，学生参与意愿低 | 数据质量差，漏填率高 |
| **标准化不足** | 依赖辅导员主观判断 | 评估一致性差，漏报与误报并存 |
| **响应延迟** | 高风险学生从筛查到干预周期长 | 错失最佳干预窗口 |
| **隐私顾虑** | 学生担心数据泄露、被标签化 | 瞒报（尤其自杀意念）比例高 |

### 1.3 人工智能介入的可行性与必要性

近年来，自然语言处理（NLP）和机器学习技术的快速发展为心理健康筛查提供了新的可能性：

1. **文本分析**：社交媒体和日记文本已被研究证明能反映抑郁症状（De Choudhury et al., 2013）
2. **多模态融合**：结合问卷、文本和生理信号可显著提升检测准确率（Garcia-Ceja et al., 2018）
3. **轻量级模型**：在小样本（<5000）场景下，Logistic Regression 等传统方法优于黑箱深度学习（Christodoulou et al., 2019）
4. **可解释性需求**：医疗领域要求模型决策过程可解释，SHAP/LIME 等工具提供了可行性

### 1.4 本研究的定位

本研究旨在构建一个**辅助筛查与预警系统**——定位为"体温计"而非"诊断仪"，核心目标是：

- 帮助心理咨询师/辅导员**高效甄别**潜在高风险学生
- 提供**可解释的风险评估**，而非黑箱输出
- 在准确性与安全性之间建立**多重保障机制**
- 确保系统**可部署、可监控、可审计**

## 2. 研究目标与问题定义

### 2.1 核心研究问题

本研究围绕以下四个核心问题展开：

**RQ1: 如何在输入数据质量参差不齐的情况下保证评估有效性？**
> 现实场景中，不同学生提供的输入数据完整性差异巨大：有人能填写完整问卷，有人只愿输入几段文字，有人仅完成焦虑量表。系统需要具备自适应能力。

**RQ2: 如何在小样本条件下训练具有临床参考价值的风险评估模型？**
> 真实标注数据稀缺（<5000 样本），且类别不平衡严重（高危样本占 10-20%）。传统深度学习容易过拟合。

**RQ3: 如何确保系统对危机表达（自杀意念）的敏感性，避免漏判？**
> 心理健康场景下，漏判一个高风险学生的代价远大于误判多个低风险学生。系统需要特殊的"安全优先"设计。

**RQ4: 如何使系统具备可解释性、可监控性和可审计性，满足医疗伦理要求？**
> 黑箱模型在医疗领域不可接受，系统必须能解释"为什么判定该生为高风险"。

### 2.2 系统功能需求

| 角色 | 核心需求 | 关键功能 |
|:---|:---|:---|
| **学生 (User)** | 快速自评 | 文本自由输入、结构化问卷填写、即时风险反馈 |
| **咨询师 (Counselor)** | 高效复核 | 预警列表、详情页查看、复核意见记录、历史追溯 |
| **管理员 (Admin)** | 全局监控 | Dashboard 总览、引擎健康快照、路由分布统计、审计导出 |
| **系统 (System)** | 安全保障 | 危机检测 override、模型回退、只升不降保护、审计日志 |

### 3.2 数据来源与数据处理

本研究使用三层次数据体系，合计约 5000+ 有效样本：

| 数据层 | 来源 | 样本量 | 维度 | 用途 |
|:---|:---|:---:|:---:|:---|
| **真实公开数据** | PHQ-9 5th Edition | ~1500 | 结构化问卷 | v1.23 外部验证 |
| **真实公开数据** | Student Depression Dataset | ~2800 | 18 特征 | v1.20 训练、v1.21/v1.25/v1.26 训练 |
| **合成数据** | 启发式规则生成 | ~1000 | 12 特征 | v1.20 初始训练 |
| **文本语料** | 标注合成 + 公开语料 | ~2000 | 自由文本 | 文本模型训练 |

**数据处理流程**:

```
原始数据 → 缺失值处理(训练集中位数) → 异常值裁剪(P1-P99)
         → 特征标准化(StandardScaler) → 平衡化(SMOTE, 比例 0.8:1)
         → 划分(Train 70% / Val 15% / Test 15%) → 训练/评估
```

**关键数据处理原则**:
- 所有缺失值填充、异常值裁剪、标准化参数均**仅在训练集上拟合**，再应用于验证集/测试集
- SMOTE 合成比例控制在 0.8:1 以内，防止过度平衡导致决策边界失真
- 固定随机种子（`random_state=42`），确保实验可复现

### 3.3 模型选择策略

考虑到样本量限制（<5000）和可解释性需求，本研究采用**传统 ML + 轻量 DL 混合策略**：

| 决策维度 | 选择 | 原因 |
|:---|:---|:---|
| **核心模型** | Logistic Regression (LR) | 在小样本场景中表现优于复杂模型；天然可解释 |
| **辅助模型** | Random Forest (RF) | 作为对比基线，评估非线性模型增益 |
| **文本模型** | TF-IDF + LR / 轻量 DNN | 嵌入维度 128，参数量 < 5000 |
| **融合模型** | Stacking Ensemble | 多模态特征级融合 + 模型级加权 |
| **可解释性** | SHAP + Permutation Importance | 模型无关，满足医疗可解释性要求 |
| **拒绝使用** | Transformer/BERT 大规模预训练模型 | 参数量过大（>100M），在 <5000 样本场景下严重过拟合 |

### 4.2 路由决策引擎设计

路由决策引擎是系统的核心入口，根据输入完整性自动分流：

```
                    ┌─────────────┐
                    │  接收输入    │
                    └──────┬──────┘
                           │
                           ▼
                   ┌───────────────┐
                   │ Crisis 检测    │───YES──→ ⚠ Crisis Override → risk ≥ 3
                   └───────┬───────┘
                           │ NO
                           ▼
                   ┌───────────────┐
                   │ 结构化特征 ≥   │───YES──→ 📊 structured
                   │ 10/12?        │           (v1.20 LR, 12特征, default)
                   └───────┬───────┘
                           │ NO
                           ▼
                   ┌───────────────┐
                   │ 轻特征 ≥ 7/9? │───YES──→ 📊 lite
                   │               │           (v1.25 LR, 9特征, 阈值0.40)
                   └───────┬───────┘
                           │ NO
                           ▼
                   ┌───────────────┐
                   │ 仅焦虑量表 ≥  │───YES──→ 📊 anxiety_only
                   │ 4/7?          │           (焦虑专项评估)
                   └───────┬───────┘
                           │ NO
                           ▼
                   ┌───────────────┐
                   │ 仅有文本输入? │───YES──→ 📝 text_fusion
                   │               │           (文本+融合后备)
                   └───────┬───────┘
                           │ NO
                           ▼
                   ┌───────────────┐
                   │ 输入不足      │───YES──→ ⚡ insufficient
                   │               │           (提示补充 + 启发式警告)
                   └───────────────┘
```

**路由设计原则**:
- **确定性**: 相同输入始终路由到相同路径
- **安全性优先**: Crisis 检测在任何路由决策之前执行
- **透明性**: 路由决策过程可追溯（`route_trace` 字段）

## 5. 模型演进与实验结果分析

### 5.1 模型演进路线与关键决策

本研究从 v1.0 原型开始，历经多次重大模型版本迭代。以下聚焦 **4 个关键模型版本**：

#### 5.1.1 v1.20 — 结构化逻辑回归模型 (Structured LR)

**核心改进**: 从合成数据迁移到真实数据集（Student Depression Dataset），实现从"模拟"到"真实"的关键跨越。

**训练详情**:
- 数据集: Student Depression Dataset + 合成数据补充 (~3000 样本)
- 特征维度: 12 维（age, cgpa, stress_level, sleep_duration, social_support, financial_pressure, family_history, academic_pressure, exercise_frequency, anxiety, panic_attack, treatment_seeking）
- 数据划分: Train 70% / Val 15% / Test 15%
- 类别平衡: SMOTE (比例 0.8:1)
- 模型配置: `LogisticRegression(max_iter=5000, class_weight='balanced', random_state=42)`

**性能指标**:

| 指标 | 单次测试集 | 5-Fold CV (mean ± std) |
|:---|:---:|:---:|
| Accuracy | **0.9833** | 0.9827 ± 0.0024 |
| F1-Score | **0.9875** | 0.9870 ± 0.0018 |
| ROC-AUC | **0.9991** | 0.9991 ± 0.0003 |
| Precision | 0.9900 | — |
| Recall | 0.9850 | — |

**风险等级映射**:
```
risk_score ∈ [0, 0.5)   → Level 1 (低风险 / LOW)
risk_score ∈ [0.5, 0.75) → Level 2 (中风险 / MODERATE)
risk_score ∈ [0.75, 1.0] → Level 3 (高风险 / HIGH)
```

**结论**: ✅ 升级为 `default`（系统默认模型），成为 structured 路由的默认模型。

#### 5.1.2 v1.21 — 多分类实验（已禁用）

**实验动机**: 尝试将 binary 升级为 multiclass（LOW/MODERATE/HIGH 三类直接预测）。

**结果**: ❌ 三类 F1 严重不均衡：LOW 0.95 vs HIGH 0.12。高风险样本大量误判为中等风险 → **临床不可接受**。决策边界不稳定，泛化性差。

**决策**: 禁用 v1.21，回退到 v1.20 binary + 阈值分级方案。教训：在小样本场景下，binary 分类可靠性远高于 multiclass。

#### 5.1.3 v1.23 — 外部风险模型 (External LR)

**核心改进**: 引入独立外部数据集（PHQ-9 5th Edition ~1500 样本）进行跨数据集验证，评估模型泛化能力。

| 指标 | 值 |
|:---|:---:|
| ROC-AUC | **0.9131** |
| F1-Score | 0.8589 |
| Recall | **0.8733** |
| Precision | 0.8452 |

**关键发现**: AUC 从内部测试 0.9991 降到外部 0.9131（-8.6%），暴露了内部测试的乐观偏差。Recall 0.8733 意味着漏报率 12.67% — 在安全边际内但仍有改进空间。

**结论**: ✅ 升级为 `limited_active`，供外部验证和消融实验使用。

#### 5.1.4 v1.25 + v1.26 — Lite 轻特征模型

**核心改进**: 将 12 特征精简至 9 特征，降低对完整问卷的依赖，配合阈值优化实现"轻量化但安全"的平衡。

**v1.25 核心特征（9维）**: stress_level, sleep_duration, social_support, financial_pressure, family_history, anxiety, panic_attack, treatment_seeking, exercise_frequency

**v1.25 原始指标（阈值=0.50）**:

| 指标 | 值 |
|:---|:---:|
| ROC-AUC | 0.9380 |
| Recall | 0.6667 |
| Specificity | 0.9542 |
| F1-Score | 0.7273 |
| Brier Score | 0.0710 |

**问题**: 默认 0.50 阈值下 Recall 仅 0.6667 — **漏报率高达 33.3%**，在心理健康场景中不可接受。

**v1.26 阈值优化**: 经系统性阈值扫描（threshold sweep），将决策阈值从 0.50 下调至 0.40。

| 指标 | v1.25 (阈值=0.50) | v1.26 (阈值=0.40) | 变化 |
|:---|:---:|:---:|:---:|
| **Recall** | 0.6667 | **0.7692** | +15.4% ↑ |
| **Specificity** | 0.9542 | **0.9413** | -1.3% ↓ |
| F1-Score | 0.7273 | 0.7895 | +8.5% ↑ |
| AUC | 0.9380 | 0.9380 | — (模型不变) |

**阈值优化分析**:

```
Recall vs Specificity Trade-off:

    Recall
    1.0 ┤
    0.9 ┤                                           ○ (0.30)
    0.8 ┤                    ● (0.40) ← 选中
    0.7 ┤        ▲ (0.50) ← 原始
    0.6 ┤
    0.5 ┤
        └──┬─────┬─────┬─────┬─────┬─────┬────→ Specificity
          0.6   0.7   0.8   0.9   1.0

● = 选取的平衡点: Recall 0.7692, Specificity 0.9413
   → 漏报率从 33.3% 降至 23.1% (↓31%)
   → 误报率仅从 4.6% 升至 5.9% (↑28%)
```

**v1.26 Go/No-Go 判定**:

| 判定条件 | 阈值要求 | 实际值 | 判定 |
|:---|:---:|:---:|:---:|
| Lite Recall | ≥ 0.75 | **0.7692** | ✅ |
| Specificity | ≥ 0.65 | **0.9542** | ✅ |
| AUC | ≥ 0.88 | **0.9380** | ✅ |
| Brier Score | ≤ 0.12 | **0.0710** | ✅ |
| 路由稳定性 | structured不误分到lite | **7/8 场景通过** | ✅ |
| 阈值文档 | 明确文档化 | **已完成** | ✅ |
| 回退方案 | 可安全回退到 0.50 | **已就绪** | ✅ |

**结论**: ✅ GO — 升级为 `limited_active`，成为 lite 路由默认模型。

## 6. 核心创新点

### 6.1 创新点一：基于输入质量的分级路由架构

**问题**: 传统系统通常要求用户完成固定问卷，但现实中输入完整性差异巨大。

**方案**: 设计四级自适应路由，根据特征覆盖率自动选择最优模型：

```
输入质量梯度 → 路由策略
─────────────────────────────────
高质量 (12/12特征)  → structured   (v1.20, default)
中质量 (9/12特征)   → lite         (v1.25+v1.26, 阈值0.40)
低质量 (4/7焦虑项)  → anxiety_only (专项评估)
极低质量 (仅文本)    → text_fusion  (文本模型)
无有效输入           → insufficient (提示补充)
```

**创新价值**:
- 与传统"全或无"模式相比，覆盖广度提升 **3-5倍**
- 路由分布统计为系统运营提供数据驱动优化依据
- 路由收敛测试（v1.26）验证: structured 路径从不误分到 lite（7/8 场景）

### 6.2 创新点二：安全优先的危机覆盖机制

**问题**: 传统 ML 系统对罕见但致命的危机表达（自杀意念）缺乏专门处理。

**方案**: 三阶段安全检测链：

```
第一阶段 (规则)          第二阶段 (语义)        第三阶段 (覆盖)
┌──────────────┐      ┌──────────────┐      ┌──────────────┐
│ 10个关键词匹配 │  →   │ 文本情感分析   │  →   │ 强制风险 ≥ 3  │
│ 自杀/自残/…   │      │ 负向极性 + 紧  │      │ 触发 crisis   │
│ 直接匹配      │      │ 迫度评分      │      │ override 标记  │
└──────────────┘      └──────────────┘      └──────────────┘
```

**关键设计**:
- **Crisis Override**: 无论模型预测结果如何，一旦触发危机检测，风险等级强制设置为 ≥ 3（HIGH）
- **只升不降**: 危机 override 只能提升风险等级，不能降低
- **人工复核提醒**: 前端醒目提示"Crisis Alert"，要求咨询师立即关注
- **审计记录**: 所有 override 事件记录完整审计链

### 6.3 创新点三：可解释的模型输出

**问题**: 深度学习模型在医疗领域的"黑箱"特性是部署的最大障碍。

**方案**: 三层次可解释性框架：

```
第一层: 全局特征重要性     → SHAP Summary Plot (所有预测)
第二层: 单样本解释          → SHAP Waterfall / Force Plot (单个预测)
第三层: 路由决策解释        → route_trace 字段 (为什么选这个路由)
```

**技术实现**: SHAP 模型无关解释 + Permutation Importance 双重验证。

**示例输出**（高风险学生）:
```
Top contributing factors:
  + stress_level:         0.245 (压力水平高 → 风险↑)
  + anxiety:              0.198 (焦虑评分高 → 风险↑)
  - social_support:      -0.087 (社会支持好 → 风险↓)
  + family_history:       0.065 (家族史阳性 → 风险↑)
```

### 6.4 创新点四：模型生命周期治理体系

**问题**: 多模型系统中如何安全地引入、升级、回退模型？

**方案**: 五阶段生命周期状态机 + 自动回退机制：

```
candidate → experimental → limited_active → default
                ↓              ↓               ↓
             废弃路径 ← deprecated → disabled
```

**治理价值**:
- 9 个注册模型中仅 4 个活跃使用，避免模型膨胀
- Shadow 部署支持新模型在真实流量下无声验证
- 模型健康快照实时监控加载状态和预测统计

### 6.5 创新点五：五级容错回退机制

系统在多层级的失败场景下确保可用性：

```
Level 0: 正常推理 (Normal)
        ↓ 失败
Level 1: PyTorch/依赖不可用 → 回退到 scikit-learn
        ↓ 失败
Level 2: 特定模型加载失败 → 回退到备选模型
        ↓ 失败
Level 3: 模型预测异常值 (NaN/Inf) → 回退到启发式规则
        ↓ 失败
Level 4: 完全空输入 → 安全默认值 (insufficient 警告)
```

## B. 答辩问答准备（扩展版）

### Q1: 请简要介绍你的研究动机和创新点。

**A**: 研究动机来自三个观察：(1) 高校心理健康筛查覆盖率不足、效率低；(2) AI 在心理健康领域有潜力但面临小样本、可解释性、安全性三大瓶颈；(3) 现有工具多为"全或无"的自评量表，缺乏自适应能力。

五大创新点：
1. **分级路由架构** — 根据输入质量自适应选择模型，覆盖广度提升 3-5 倍
2. **Crisis Override** — 三阶段安全检测链，危机表达零容忍漏判
3. **可解释性输出** — SHAP 三层次解释（全局/单样本/路由）
4. **模型生命周期治理** — 五阶段状态机管理 9 个模型，4 个活跃
5. **五级容错回退** — 从 DL→ML→规则→默认值的渐进降级

### Q2: 为什么选择 Logistic Regression 作为核心模型，而不是深度学习？

**A**: 这是基于数据量和场景需求的数据驱动决策：

1. **数据量限制**: 训练样本 <5000，Transformer/BERT 等模型参数量 >100M，在此规模下极易过拟合（Christodoulou et al., 2019）。

2. **可解释性需求**: LR 系数天然可解释（每个特征的风险方向），而 DNN/Transformer 是黑箱。

3. **实证验证**: 
   - v1.20 LR: F1=0.9875, AUC=0.9991
   - v1.21 RF: F1=0.6829 → 非线性模型改善有限
   - v1.21 Multiclass DNN: 过拟合 → 被禁用

4. **部署简单**: 无 GPU 依赖，推理延迟 < 1ms。

### Q3: v1.20 的 Accuracy 0.9833 和 AUC 0.9991 这么高，是否存在过拟合？

**A**: 确实存在内部测试偏高的问题。v1.23 外部验证 AUC 从 0.9991 降至 0.9131（降幅 8.6%），但 0.9131 仍属良好水平。我们通过外部验证校准预期，并以 Recall（≥0.75）而非 Accuracy 作为 Go/No-Go 基准。

### Q4: Lite 模型阈值从 0.50 降到 0.40 的依据是什么？

**A**: 场景化成本权衡：**漏报代价（可能自杀）>> 误报代价（多复核一次）**。阈值 0.50→0.40 使 Recall 从 0.6667→0.7692 (+15.4%, 漏报降低 31%)，Specificity 仅从 0.9542→0.9413 (-1.3%)。用"Elbow Method"选 0.40 为 Recall 增长率下降的拐点。

### Q5: Crisis Override 如何避免误触发？

**A**: 三阶段设计：(1) 10 个危机关键词**词边界精确匹配**；(2) **否定句筛查**（"不是想死"→不触发）和**引述识别**（"他说过想死"→降级为间接表达）；(3) 语义检测需同时满足负向极性>0.7 AND 紧迫度>0.5。

### Q6: 系统能用于真实临床诊断吗？

**A**: **明确不能**。本系统定位为"体温计"（筛查），非"诊断仪"。类比：体温计测出 38°C → 建议看医生，但不能说"你感冒了"。系统已标注"仅供参考，不能替代专业医疗诊断"，高风险仅建议咨询心理咨询师。

### Q7: 数据来源和伦理如何处理？
