## 一、监控目标

v1.22 监控目标是保证结构化风险输出在实验真实模型、默认模型和 fallback 之间保持可观测、可回滚、可审计。

重点监控：

1. 模型输出分布是否异常；
2. High/Critical 风险占比是否漂移；
3. fallback 使用率是否升高；
4. 输入特征质量是否下降；
5. 实验真实模型与默认模型差异是否扩大；
6. 推理性能是否满足服务要求。

## 四、Fallback 监控

| 指标 | 正常范围 | Warn | Alert |
|------|----------|------|-------|
| 模型文件缺失回退率 | 0% | >1% | >5% |
| 特征缺失回退率 | <5% | >10% | >25% |
| 异常值回退率 | <1% | >5% | >10% |
| 总 fallback 使用率 | <5% | >10% | >25% |

fallback 使用率升高通常说明：

- 输入表单字段缺失；
- 前后端特征契约不一致；
- 模型 artifact 加载失败；
- 数据编码变化未同步；
- 新版本部署缺少 scaler 或 feature_names 文件。

## 六、模型差异监控

v1.22 建议并行记录默认模型和实验真实模型输出：

| 指标 | 用途 |
|------|------|
| `default_risk_score` | 当前业务主输出 |
| `experimental_real_score` | 真实 LR 参考输出 |
| `score_delta` | 两模型差异 |
| `level_delta` | 风险等级差异 |
| `model_version` | 审计模型来源 |
| `fallback_used` | 判断是否回退 |

建议告警：

- `abs(score_delta) > 40` 的样本占比超过 20%；
- 实验模型 High/Critical 占比是默认模型 2 倍以上；
- 默认模型和实验模型在 Critical 上长期不一致。

## 八、运营复核指标

如果 v1.22 接入人工复核，应新增：

| 指标 | 说明 |
|------|------|
| High/Critical 复核确认率 | 高风险预测中被人工确认需关注的比例 |
| Critical 误报率 | Critical 中被确认无需关注的比例 |
| 漏报反馈数 | 低风险但后续被确认高风险的数量 |
| 平均复核时长 | 高风险任务从产生到处理的时间 |
| 升级处理率 | 进入危机流程或人工干预的比例 |

---

## 九、结论

v1.22 监控基线应围绕模型输出、输入质量、fallback、模型差异和人工复核闭环建立。

在真实模型尚未成为默认模型前，最重要的不是追求自动化替换，而是积累线上分布证据，为下一轮真实数据校准和模型升级提供依据。

### 10.2 实测样本差异分析

| 样本 | Default Score | Default Level | Exp Score | Exp Level | Delta |
|------|:------------:|:------------:|:---------:|:---------:|:-----:|
| 高1 | 100.0 | 4 (严重) | 100.0 | 4 (严重) | 0.00 |
| 中2 | 35.8 | 2 (中度) | 98.41 | 4 (严重) | **+62.62** ⚠️ |
| 低3 | 0.0 | 0 (无) | 1.80 | 0 (无) | +1.80 |
| 高4 | 100.0 | 4 (严重) | 99.96 | 4 (严重) | -0.04 |

> ⚠️ **样本2**: 默认模型判定为中度风险(35.8)，实验真实模型判定为严重(98.41)。差异+62.62 触发 `abs(score_delta) > 40` 告警。该样本特征为中等压力+低社会支持，真实LR模型对此类组合高度敏感。

### 10.4 下一步监控行动

1. **上线后前两周**：密切关注 `high_critical_ratio` 和 `mean_abs_delta`
2. **样本2类情况**：如 `abs(score_delta) > 40` 占比 > 20%，需评估真实模型是否过度敏感
3. **Fallback 率**：保持 < 5%，如升高需检查模型文件部署
4. **人工复核**：建议对实验模型高/默认模型中度的样本优先复核
