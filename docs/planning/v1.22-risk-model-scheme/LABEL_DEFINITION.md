## 一、标签体系目标

v1.22 标签体系的目标是统一二分类风险识别、五级风险展示和线上告警逻辑，避免继续使用无法可靠训练的独立五级模型。

v1.22 采用以下原则：

1. 二分类标签继续以 Depression 字段作为 ground truth；
2. 五级风险不再作为独立监督学习主目标；
3. 五级风险由二分类概率、风险分数和业务阈值映射得到；
4. 派生标签必须显式标注为非临床标签；
5. 所有标签定义必须支持回滚和可解释。

## 二、二分类标签定义

### 2.1 标签含义

| 编码 | 标签 | 含义 |
|------|------|------|
| 0 | No Depression | 未观察到抑郁标签 |
| 1 | Depression | 观察到抑郁标签 |

### 2.2 构造规则

```python
binary_label = 1 if Depression == "Yes" else 0
```

### 2.3 当前分布

| 标签 | 样本数 | 比例 |
|------|--------|------|
| 0 | 806 | 80.6% |
| 1 | 194 | 19.4% |

### 2.4 训练注意事项

- 使用 `class_weight='balanced'`；
- 评估时优先关注 Recall、ROC-AUC 和 AUPRC；
- 心理健康预警场景中，漏报成本高于误报成本；
- 不以单一 Accuracy 作为上线判断依据。

## 三、五级风险标签的 v1.22 定位

v1.21 曾通过加权规则构造五级派生标签：

```python
score = 0
if Depression == 'Yes': score += 2
if Anxiety == 'Yes': score += 1
if Panic_Attack == 'Yes': score += 2
if Treatment_Seek == 'Yes': score += 1
if Stress_Level >= 8: score += 1
```

该规则可用于审计和离线分析，但 v1.22 不建议继续将其作为独立五级模型的主要训练标签，原因是：

1. 标签本身由多个输入字段派生，容易造成数据泄漏；
2. 移除泄漏字段后，剩余 10 个基础特征预测力不足；
3. v1.21 五级模型在测试集上 Accuracy 仅 13%-32%；
4. Level 2、Level 4 样本量不足；
5. 五级派生标签不是临床专业标注。

## 四、v1.22 五级风险展示定义

v1.22 推荐采用“二分类概率 → 风险分数 → 五级等级”的展示标签。

### 4.1 概率分档

| Depression Probability | 风险等级 | 编码 | 展示名 |
|------------------------|----------|------|--------|
| p < 0.15 | None | 0 | 无明显风险 |
| 0.15 ≤ p < 0.35 | Mild | 1 | 轻度风险 |
| 0.35 ≤ p < 0.65 | Moderate | 2 | 中度风险 |
| 0.65 ≤ p < 0.85 | High | 3 | 高风险 |
| p ≥ 0.85 | Critical | 4 | 极高风险 |

### 4.2 风险分数映射

风险分数范围为 0-100，建议按照分段函数压缩高概率区间，避免因未校准概率导致过度告警。

```python
if p < 0.05:
    risk_score = p * 200
elif p < 0.15:
    risk_score = 10 + (p - 0.05) * 100
elif p < 0.35:
    risk_score = 20 + (p - 0.15) * 100
elif p < 0.65:
    risk_score = 40 + (p - 0.35) * 83
elif p < 0.85:
    risk_score = 65 + (p - 0.65) * 100
else:
    risk_score = 85 + (p - 0.85) * 100
```

### 4.3 分数到等级

| Risk Score | 风险等级 |
|------------|----------|
| 0-19 | None |
| 20-39 | Mild |
| 40-59 | Moderate |
| 60-79 | High |
| 80-100 | Critical |

## 五、标签使用规范

| 场景 | 标签来源 | 是否允许 |
|------|----------|----------|
| 二分类模型训练 | Depression | 允许 |
| 二分类模型评估 | Depression | 允许 |
| 独立五级模型训练 | 派生 Five-Level | 不推荐 |
| 前端风险等级展示 | 概率分档 | 推荐 |
| CrisisEvent 触发 | High/Critical + 业务规则 | 推荐 |
| 临床诊断 | 任一模型标签 | 禁止 |

---

## 六、风险文案规范

v1.22 标签必须使用“风险提示”语义，不得使用“诊断”语义。

推荐文案：

- “系统识别到较高心理健康风险”；
- “建议进行进一步评估”；
- “该结果仅用于辅助筛查”；
- “不能替代专业诊断”。

禁止文案：

- “用户患有抑郁症”；
- “模型诊断为重度抑郁”；
- “该用户必然存在危机”。

---

## 七、v1.22 标签结论

v1.22 的标签体系应从“训练五级标签”转向“解释二分类概率并稳定映射为五级风险”。

最终推荐：

- **训练目标**: Depression 二分类；
- **展示目标**: 五级风险等级；
- **告警目标**: High/Critical 风险合并监控；
- **临床定位**: 非诊断，仅辅助筛查。
