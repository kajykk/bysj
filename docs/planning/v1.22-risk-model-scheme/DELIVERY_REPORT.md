## 一、执行总结

v1.22 基于 v1.21 真实数据建模结论，完成了结构化心理风险模型的方案收敛。

核心结论：

> 当前数据条件下，不应继续推进独立五级分类模型上线；应采用“稳定默认模型 + 真实二分类实验模型 + 概率分档五级风险 + fallback 兜底 + 监控闭环”的组合方案。

v1.22 的价值在于从“模型训练”转向“可上线方案设计”，明确了数据边界、标签边界、模型选择、校准方式、解释性策略和监控基线。

- 不启用 v1.21 五级独立模型。

## 六、上线建议

### Phase 1：文档与配置准备

- 确认 v1.22 报告集；
- 确认模型版本配置；
- 确认多分类模型禁用；
- 确认 fallback 可用；
- 确认前端文案为“风险提示”而非“诊断”。

### Phase 2：实验路径接入

- 并行输出 v1.21 Real Binary LR 概率；
- 不改变主风险等级；
- 记录实验分数和默认分数差异；
- 后台展示实验字段。

### Phase 3：监控观察

- 连续观察至少两周；
- 统计 High/Critical 占比；
- 统计 fallback 使用率；
- 收集人工复核反馈；
- 评估是否需要调整阈值。

### Phase 4：后续升级

- 补充 PHQ-9/GAD-7 或咨询师复核标签；
- 增加真实样本规模；
- 重新评估概率校准器；
- 重新评估真实模型替代默认模型的可行性。

## 七、主要风险

| 风险 | 影响 | 缓解策略 |
|------|------|----------|
| 真实模型误报较多 | 人工复核压力上升 | 实验路径展示，不作为主触发 |
| 概率未充分校准 | 风险等级偏高 | 使用分段映射并监控分布 |
| 五级风险被误解为诊断 | 合规与伦理风险 | 明确非诊断文案 |
| 输入特征缺失 | fallback 增加 | 建立输入质量监控 |
| 默认模型仍基于合成规则 | 真实性不足 | 保留真实模型并行观测，推动数据采集 |

## 八、交付结论

v1.22 达成 **Scheme Go**：方案可进入配置检查、灰度观测和后续实现阶段。

最终建议如下：

1. 保持 v1.20 / fallback 为默认主路径；
2. 保留 v1.21 Real Binary LR 为实验参考；
3. 禁用独立五级分类模型；
4. 使用概率分档生成五级风险；
5. 建立监控和人工复核闭环；
6. 后续通过更高质量标签和更多真实样本推进 v1.23。
