## 一、方案目标

v1.22 二分类模型方案的目标不是重新训练并宣称更高指标，而是基于 v1.21 的真实评估结果，明确：

1. 哪个二分类模型可以继续作为实验参考；
2. 为什么不能直接替代 v1.20 默认模型；
3. v1.22 如何将该模型纳入展示、监控和灰度验证；
4. 后续如何扩充到更可信的真实数据版本。

## 三、v1.22 方案判断

### 3.1 推荐保留的实验模型

v1.22 推荐继续保留 **v1.21 Real Binary LogisticRegression** 作为实验性参考模型，理由如下：

1. Recall 达到 0.9130，适合高召回预警场景；
2. ROC-AUC 0.9401，说明模型区分能力较强；
3. 尽管 Precision 较低，但误报比漏报更容易通过人工复核吸收；
4. 在心理健康风控场景中，漏掉高风险用户的代价更高。

### 3.2 不推荐作为默认主模型的原因

| 问题 | 说明 |
|------|------|
| F1 不达标 | 0.6000 低于既定目标 0.85 |
| 精度偏低 | 会产生较多误报 |
| 概率未充分校准 | 高概率区间存在高估 |
| 数据规模有限 | 主数据集仅 1,000 样本 |
| 关键特征共病性较强 | 模型较依赖 Anxiety、Panic、Treatment 这类近邻症状特征 |

## 四、v1.22 的模型使用方式

### 4.1 推荐定位

| 用途 | 说明 |
|------|------|
| 实验分数输出 | 作为 ReviewTask 或管理后台的辅助信号 |
| 阈值分档输入 | 将 `predict_proba` 映射为五级风险 |
| 监控对比基线 | 用于观察真实模型与 heuristic 模型的分布差异 |
| 灰度验证 | 限定环境开启，不直接影响主业务决策 |

### 4.2 不推荐定位

- 不直接替换 v1.20 默认线上模型；
- 不单独作为 CrisisEvent 的唯一触发依据；
- 不作为临床严重程度判断工具；
- 不在缺少人工复核时单独给出高危结论。

---

## 五、阈值策略建议

基于 v1.21 的结果，v1.22 建议采用偏保守的阈值分档，而不是简单使用 0.5 二值阈值。

| 概率区间 | 建议动作 |
|----------|----------|
| p < 0.15 | 低风险展示 |
| 0.15 ≤ p < 0.35 | 轻度提醒 |
| 0.35 ≤ p < 0.65 | 中度关注 |
| 0.65 ≤ p < 0.85 | 高风险复核 |
| p ≥ 0.85 | 极高风险复核/升级处理 |

推荐原因：

1. 0.4-0.6 区间在真实测试中校准较差；
2. 直接按 0.5 做分界会过于粗糙；
3. 前台展示和后台复核应采用不同敏感度；
4. 多阈值机制更适合风险管理而非硬分类任务。

## 六、v1.22 方案落地建议

### 短期

- 保留 LogisticRegression 作为实验模型；
- 将输出概率接入统一 risk_score 映射；
- 在后台显示 `experimental_real_model_score`；
- 与 v1.20 heuristic/fallback 同步观测至少两周；
- 不改变默认主路由。

### 中期

- 引入更多真实标签样本；
- 进行阈值再校准；
- 引入更细粒度问卷或文本特征；
- 比较“结构化二分类 + 文本情绪模型”的融合效果。

---

## 七、结论

v1.22 的二分类模型方案应采用“**保留真实 LR 作为实验风险参考，不替代默认模型；以高召回概率输出支撑五级风险分档和人工复核**”的策略。

这比继续追求一个尚未达标的单模型上线更稳妥，也更符合 v1.21 已暴露出的真实数据边界。
