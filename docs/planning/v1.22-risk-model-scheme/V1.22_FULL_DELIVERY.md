## 一、项目定位

v1.22 为**方案收敛迭代**——不训练新模型，而是基于 v1.21 的真实数据实验结论与 v1.20 的 Synthetic 基线，输出可上线的模型架构方案。

**核心决策**：维持 v1.20 Synthetic LR 为默认主模型，v1.21 Real Binary LR 为实验并行参考，禁用五级多分类模型（改用概率分档）。

## 五、Phase 3 — 监控观察 ✅

### 5.1 监控基础设施

已在 `model_engine.py` 中内置 8 项实时指标，通过 `get_metrics_snapshot()` 暴露：

```json
{
  "monitoring": {
    "structured_total":         "预测总数",
    "high_critical_ratio":      "High/Critical 占比",
    "high_critical_count":      "High/Critical 次数",
    "fallback_ratio":           "Fallback 使用率",
    "fallback_count":           "Fallback 次数",
    "experimental_hit_ratio":   "实验路径命中率",
    "input_quality": {
      "complete": "完整输入",
      "partial":  "部分缺失",
      "poor":     "严重缺失"
    },
    "score_delta_recent": {
      "count":         "最近差异样本数",
      "mean_abs_delta": "平均绝对差异",
      "max_abs_delta":  "最大差异"
    }
  }
}
```

### 5.2 告警规则

| 条件 | 级别 | 动作 |
|------|:--:|------|
| `abs(score_delta) > 40` | ⚠️ WARNING | 审查该样本特征 |
| `fallback_ratio > 0.05` | 🔴 CRITICAL | 检查模型文件部署 |
| `high_critical_ratio > 0.60` | ⚠️ WARNING | 评估是否需要阈值调整 |
| Delta 样本占比 > 20% | 🔴 CRITICAL | 评估实验模型过度敏感性 |

## 八、后续路线图 (v1.23+)

### 8.1 v1.23 — 第一阶段推荐行动

```
Step 1: 用 aligned_features.csv (28k) 训练 LR 基线
        ↓
Step 2: 用 Mendeley PHQ-9 (682) 微调 / 验证
        ↓
Step 3: 用 mmpsy (1,275) 做外部验证 (PHQ-9 一致性)
        ↓
Step 4: 对比 v1.20 Synthetic LR / v1.21 Real / v1.23 External
        ↓
Step 5: 决策是否升级默认模型
```

### 8.2 关键决策点

| 条件 | 决策 |
|------|------|
| 外部数据 LR 在 mmpsy 上 AUC > 0.80 | ✅ 可考虑替代 v1.20 |
| 外部模型与 v1.20 差异 < ±15% | ✅ 一致性可接受 |
| 外部模型在 mmpsy 上 AUC < 0.70 | ❌ 继续 v1.20 默认 |
| PHQ-9 预测皮尔逊 r < 0.5 | ⚠️ 需更丰富的结构化特征 |
