# v1.5 迭代方案提案

> **日期**: 2026-04-28

## 1. 项目现状分析

### 1.1 已有功能 (v1.4 及之前)

| 模块 | 功能 | 状态 |
|------|------|------|
| **用户系统** | 注册/登录/认证、角色管理（用户/咨询师/管理员） | ✅ 完成 |
| **数据收集** | 生理数据（可穿戴设备）、文本数据（问卷/日记） | ✅ 完成 |
| **风险评估** | 多模态融合、XGBoost/LightGBM/MLP/BERT 模型 | ✅ 完成 |
| **预警系统** | 风险等级判定、预警触发、通知渠道 | ✅ 完成 |
| **干预系统** | 干预模板、任务分配、进度跟踪 | ✅ 完成 |
| **内容管理** | 冥想/文章/视频等心理健康内容 | ✅ 完成 |
| **模型治理** | 版本管理、漂移检测、回退机制、灰度发布 | ✅ 完成 |
| **前端界面** | 用户/咨询师/管理员 Dashboard、主题切换、国际化 | ✅ 完成 |

### 1.2 技术栈

- **前端**: Vue 3 + Vite + Pinia + Element Plus + Vue I18n
- **后端**: FastAPI + SQLAlchemy + Pydantic + JWT
- **ML**: scikit-learn + XGBoost + LightGBM + PyTorch + Transformers
- **数据库**: SQLite (开发) / PostgreSQL (生产)

### 1.3 当前痛点

1. **前端性能**: 无虚拟列表、无懒加载，大数据量时卡顿
2. **实时性**: 预警依赖轮询，无实时推送
3. **可观测性**: 缺少系统监控 Dashboard、模型性能可视化
4. **数据导出**: 缺少报告生成和导出功能
5. **移动端体验**: 仅适配，未做移动端优化

## 2. v1.5 迭代方案选项

### 方案 A: 前端性能与体验优化 (推荐)

**目标**: 提升前端性能和用户体验

| 功能 | 优先级 | 描述 |
|------|--------|------|
| 虚拟列表 | P0 | 大数据量列表渲染优化 |

| 路由懒加载 | P0 | 代码分割，减少首屏加载 |

## 3. 推荐方案组合

### 推荐: 方案 A + 方案 C (前端优化 + 数据可视化)

**理由**:
1. 前端性能优化是基础，影响所有用户
2. 数据可视化是心理健康系统的核心价值（趋势洞察）
3. 两者技术风险低，可并行开发
4. 与 v1.4 的模型升级形成完整闭环（训练→监控→可视化）

**迭代名称**: `v1.5-frontend-performance-and-visualization`
