## 1. 迭代目标

将 v1.16 的"模型风险识别能力"转化为真正可运营、可处理、可追溯的安全闭环。

核心目标：

1. **上线环境验证**: 确认 v1.16 成果可在真实环境部署运行
2. **人工复核工作流**: 将 `review_required` 从字段升级为可处理的业务流程
3. **危机审计日志**: 所有 crisis 事件必须可追踪、可查询、可导出
4. **文本安全增强**: 扩展危机关键词库，提升危机识别覆盖

---

## 2. 需求范围

### 2.1 P0: 必须做

#### 2.1.1 上线环境验证

- 前端生产构建通过 (`npm run build`)
- 后端正常启动
- `/health` 和 `/health/ready` 端点正常
- 模型预测 API 冒烟测试通过
- 危机文本 API 冒烟测试通过
- 融合预测 API 冒烟测试通过

#### 2.1.2 后端人工复核工作流

- 新增复核记录表 (`review_tasks`)
- 风险预测触发时自动生成复核任务
- 咨询师可查询待复核任务列表
- 咨询师可查看复核详情（风险分数、模型贡献、触发原因）
- 咨询师可处理复核任务（标记已处理、填写备注）
- 咨询师可升级危机事件
- 复核状态流转：`pending -> in_review -> resolved -> archived`

#### 2.1.3 咨询师端复核页面

- 待复核任务列表页 (`/counselor/reviews`)
- 复核详情页 (`/counselor/reviews/:id`)
- 支持筛选：待处理、危机、高风险、已处理
- 显示模型分数、触发原因、用户历史风险记录
- 处理动作：填写备注、标记已处理、升级危机事件

#### 2.1.4 危机事件审计日志

- 新增危机事件表 (`crisis_events`)
- 自动记录危机关键词、crisis_score、触发来源
- 支持后台查询和统计
- 支持导出

### 2.2 P1: 强烈建议做

#### 2.2.1 文本安全增强

- 扩展危机关键词库（网络用语、计划性表达、求助表达）
- 增加误报过滤规则
- 增加 30+ 文本样本测试

### 2.3 P2: 可后置

- 生理模型特征扩展 (HRV、体温等) → v1.18
- 覆盖率 80% 目标 → v1.18

## 3. 用户故事

### 3.1 咨询师用户故事

**作为** 心理咨询师
**我希望** 在系统中看到模型标记的风险复核任务
**以便** 及时关注高风险用户并提供干预

**作为** 心理咨询师
**我希望** 查看用户的模型预测详情和历史风险记录
**以便** 做出更准确的判断

**作为** 心理咨询师
**我希望** 对复核任务进行处理并记录处理结果
**以便** 形成完整的干预闭环

### 3.2 管理员用户故事

**作为** 系统管理员
**我希望** 查看所有危机事件的审计日志
**以便** 了解系统的危机识别情况和处理效率

**作为** 系统管理员
**我希望** 导出危机事件报告
**以便** 进行统计分析和监管合规

## 4. 非功能需求

| 需求 | 目标 |
|---|---|
| 性能 | 复核任务列表查询 < 500ms |
| 可用性 | 危机事件记录不可丢失 |
| 安全性 | 复核数据仅限咨询师和管理员查看 |
| 扩展性 | 关键词库支持热更新 |

## 5. 验收标准

1. 前端构建成功，无报错
2. 后端启动成功，健康检查通过
3. `review_required=true` 时自动生成复核任务
4. 咨询师可查看、处理、升级复核任务
5. 危机事件自动写入审计日志
6. 危机关键词误报过滤正确
7. v1.16 核心能力回归测试通过
