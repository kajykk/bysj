## 3. P0 清零条件

上线前必须满足：

- [ ] 所有 P0 阻塞项关闭。
- [ ] 所有核心流程通过。
- [ ] 前端生产构建通过。
- [ ] 后端启动与健康检查通过。
- [ ] 数据库和模型/算法核心路径通过。
- [ ] 回滚方案完成。

---

## 4. Conditional Go 条件

若存在 P1 问题，可以 Conditional Go，但必须满足：

- [ ] P1 问题不影响核心功能。
- [ ] P1 问题不影响部署启动。
- [ ] P1 问题有明确后续修复计划。
- [ ] P1 问题已记录到交付报告。

---

## 5. No-Go 条件

出现以下任一情况，禁止上线：

- 任一 P0 阻塞项未关闭。
- 核心流程失败。
- 前端无法构建或访问。
- 后端无法启动。
- 数据库无法连接。
- 模型/算法核心结果无法返回。
- 无回滚方案。
