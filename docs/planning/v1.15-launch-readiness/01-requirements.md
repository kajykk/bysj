> **日期**: 2026-05-01

---

## 1. 背景与问题陈述

### 1.1 当前状态

v1.14 已针对 Windows 本地测试环境做了修复，但交付报告明确指出：

- Windows 下 sklearn/pandas/numpy 相关 DLL 问题仍可能导致测试崩溃。
- 覆盖率插件 `--cov=app` 仍可能触发崩溃。
- `tests/` 目录完整测试在 Windows 本地仍不稳定。
- 推荐使用 CI、Docker、WSL2 或 Linux 环境绕过本地环境问题。

因此，v1.15 不再以“覆盖率 60%→80%”作为唯一主目标，而是调整为：

> 先确保系统可以真实上线、核心功能完整可用，再用覆盖率、CI、Docker 测试支撑上线质量。

### 1.2 核心问题

当前上线前主要风险包括：

1. 核心功能是否能从前端到后端完整跑通尚需系统化验证。
2. 前后端接口、环境变量、CORS、部署配置可能存在不一致。
3. Windows 本地测试环境不稳定，不能作为唯一质量判断依据。
4. 现有 v1.15 覆盖率方案偏测试指标，无法直接回答“系统是否能上线”。
5. 缺少上线阻塞清单、部署检查清单、回滚方案和上线后验证清单。

## 2. 迭代目标

### 2.1 主目标

- [ ] 核心业务全流程可以在目标环境真实跑通。
- [ ] 前端生产构建成功，页面可访问。
- [ ] 后端服务可启动，健康检查可访问。
- [ ] 前后端 API 调用链路稳定可用。
- [ ] 数据库读写、核心业务计算、模型/算法调用可正常工作。
- [ ] 上线前冒烟测试通过。
- [ ] 形成部署、回滚、上线后验证文档。

### 2.2 支撑目标

- [ ] 建立 Docker/Linux/CI 作为后端测试主验证环境。
- [ ] 保留关键模块测试补充，但不把覆盖率 80% 作为上线阻塞条件。
- [ ] 确保前端类型检查和生产构建可以作为上线门禁。
- [ ] 建立最小 CI 质量门禁，防止核心流程回归。

### 2.3 后置目标

以下目标不阻塞 v1.15 上线，可放入 v1.16 或上线后优化：

- [ ] 后端整体覆盖率提升至 80%。
- [ ] 完整 E2E 自动化测试。
- [ ] Lighthouse 性能专项优化。
- [ ] 更完善的监控、告警和链路追踪。
- [ ] Windows 本地完整 pytest 稳定运行。

---

## 3. 范围定义

### 3.1 本轮必须覆盖

1. 核心用户路径盘点与验证。
2. 前端构建和关键页面访问验证。
3. 后端启动、健康检查和核心 API 验证。
4. 数据库初始化、连接和核心数据读写验证。
5. 模型/算法入口验证。
6. 上线环境变量和部署命令整理。
7. 上线阻塞问题清零。
8. 回滚方案与上线后检查清单。

### 3.2 本轮不优先覆盖

1. 非核心页面的体验优化。
2. 非核心 API 的完整单元测试补齐。
3. 大规模重构。
4. 覆盖率强制达到 80%。
5. 新增业务功能。

## 4. 验收标准

### 4.1 P0 上线准入标准

- [ ] 前端生产构建通过。
- [ ] 后端服务在目标环境启动成功。
- [ ] 健康检查接口返回成功。
- [ ] 核心用户流程手动验证通过。
- [ ] 核心 API 成功响应。
- [ ] 数据库读写正常。
- [ ] 核心算法/模型结果可返回。
- [ ] 关键异常输入不会导致白屏或服务崩溃。
- [ ] `.env.example` 或等价环境变量说明完整。
- [ ] `DEPLOYMENT_CHECKLIST.md` 完成。
- [ ] `ROLLBACK_PLAN.md` 完成。
- [ ] `POST_LAUNCH_CHECKLIST.md` 完成。

### 4.2 P1 质量支撑标准

- [ ] Docker/Linux 环境可运行后端关键测试。
- [ ] CI 能运行前端构建和后端关键测试。
- [ ] 覆盖率报告可以生成，结果仅作为参考。
- [ ] 关键模块测试覆盖新增或保留。

---

## 5. 决策原则

当上线速度、功能完整性和覆盖率目标发生冲突时，按以下优先级决策：

1. 核心功能真实可用。
2. 可部署、可回滚、可验证。
3. CI/Docker 支撑上线质量。
4. 覆盖率提升。
5. 非核心体验优化。
