# 上线后检查清单

> **迭代名称**: v1.15-launch-readiness  
> **用途**: 上线后快速确认系统运行状态  
> **状态**: Draft

---

## 1. 上线后 5 分钟检查

- [ ] 前端首页可访问。
- [ ] 核心页面可访问。
- [ ] 后端健康检查通过。
- [ ] 核心 API 返回成功。
- [ ] 服务日志无持续 P0 错误。
- [ ] 数据库连接正常。

---

## 2. 上线后 30 分钟检查

- [ ] 完整执行一次核心用户流程。
- [ ] 验证核心数据写入和读取。
- [ ] 验证模型/算法结果返回。
- [ ] 验证失败路径提示正常。
- [ ] 检查后端错误日志。
- [ ] 检查前端控制台错误，如可访问。

---

## 3. 上线后 24 小时检查

- [ ] 汇总用户反馈或异常报告。
- [ ] 汇总服务错误日志。
- [ ] 确认无新增 P0/P1 问题。
- [ ] 更新 `DELIVERY_REPORT.md` 的上线后状态。
- [ ] 将后续优化项写入 `NEXT_STEPS.md`。

---

## 4. 异常处理

如果检查中发现 P0 问题：

1. 立即记录问题现象。
2. 判断是否触发 `ROLLBACK_PLAN.md`。
3. 优先恢复服务可用。
4. 保留日志和现场信息。
5. 回滚或修复后重新执行 5 分钟检查。
