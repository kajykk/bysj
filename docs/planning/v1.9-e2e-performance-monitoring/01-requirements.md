# v1.9 需求文档: E2E 实跑、性能优化与监控体系

> **迭代名称**: v1.9-e2e-performance-monitoring
> **日期**: 2026-04-29
> **版本**: Round 3 Locked
> **状态**: Locked

---

## 1. 迭代定位

v1.9 是一次 **端到端验证与性能监控迭代**。基于 v1.8 建立的测试基础设施和质量门禁，本迭代目标是：
1. 让 E2E 测试真正运行起来
2. 建立性能基线并优化
3. 搭建监控告警体系

---

## 2. 核心目标

| 指标 | v1.8 实际 | v1.9 目标 | 优先级 |
|------|----------|----------|--------|
| E2E 测试可运行 | 基础设施就绪 | 核心流程可运行 | P0 |
| E2E 测试通过率 | 未验证 | >= 80% | P0 |
| Lighthouse Performance | 未统计 | >= 80 | P1 |
| 首屏加载时间 | 未统计 | < 3s | P1 |
| 错误监控接入 | 未配置 | Sentry 接入 | P1 |
| 性能监控接入 | 未配置 | Web Vitals 监控 | P2 |
| 告警机制 | 未配置 | 核心错误告警 | P2 |

---

## 3. 需求清单

### P0: E2E 测试实跑

#### R-E2E-001: Playwright 环境修复
- **描述**: 修复 E2E 测试运行环境
- **验收标准**: `npx playwright test` 可执行
- **依赖**: v1.6 已创建 8 个 spec 文件

#### R-E2E-002: 核心流程 E2E 测试
- **描述**: 运行并修复登录、评估、预警核心流程
- **验收标准**: 3 个核心流程测试通过

#### R-E2E-003: E2E 测试 CI 集成
- **描述**: GitHub Actions 中运行 E2E 测试
- **验收标准**: CI 中 E2E 步骤可执行

### P1: 性能优化

#### R-PERF-001: Lighthouse 基线
- **描述**: 运行 Lighthouse 并记录基线
- **验收标准**: 产出 Lighthouse 报告

#### R-PERF-002: 首屏加载优化
- **描述**: 优化首屏加载时间
- **验收标准**: FCP < 1.8s, LCP < 2.5s

#### R-PERF-003: Chunk 优化
- **描述**: 优化大 chunk 分割
- **验收标准**: 单个 chunk < 500KB

### P1: 监控体系

#### R-MON-001: Sentry 错误监控
- **描述**: 接入 Sentry 错误追踪
- **验收标准**: 前端错误可上报

#### R-MON-002: Web Vitals 监控
- **描述**: 接入 Web Vitals 性能监控
- **验收标准**: CLS、FID、LCP 可采集

### P2: 告警机制

#### R-ALERT-001: 核心错误告警
- **描述**: 配置核心错误告警规则
- **验收标准**: 500 错误、登录失败可告警

---

## 4. 非功能需求

- **可维护性**: 监控配置需文档化
- **可扩展性**: 告警规则支持动态调整
- **稳定性**: 监控系统自身需高可用

---

## 5. 风险与假设

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| Playwright 环境配置复杂 | E2E 延迟 | 使用 Docker 统一环境 |
| Sentry 服务可用性 | 监控中断 | 配置备用监控方案 |
| 性能优化回归 | 功能异常 | 优化前建立基线 |
