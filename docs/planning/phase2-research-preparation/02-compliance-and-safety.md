# Phase 2 合规与安全框架

> 本文档面向 DWS 心理健康风险评估系统 Phase 2，整合以下四大支柱：
> 1. DPIA（个人信息保护影响评估）草案
> 2. 危机升级 SOP（标准作业程序）
> 3. 人工复核 SLA（服务等级协议）
> 4. 误报/漏报报告模板
>
> 背景：系统已完成 P1-5 埋点隐私闭环（7 事件类型白名单、元数据键白名单、90 天保留期、同意/撤回机制、OperationLog 审计），具备 PII 加密（Fernet）、路径遍历防护与 TOCTOU 防护。本文档在此基础上完善数据合规与安全响应流程。

### 1.3 风险评估

#### 风险 1：敏感健康数据泄露

| 维度 | 内容 |
|------|------|
| 可能性 | 低 |
| 影响 | 高 |
| 风险等级 | 高 |
| 现有措施 | Fernet 加密、RBAC、OperationLog 审计 |
| 残余风险 | 中 |
| 改进建议 | ① 密钥定期轮换（建议 90 天）；② 数据库备份加密；③ 访问频率监控与异常告警 |

#### 风险 2：学生身份暴露

| 维度 | 内容 |
|------|------|
| 可能性 | 中 |
| 影响 | 高 |
| 风险等级 | 高 |
| 现有措施 | 角色隔离、咨询师绑定机制 |
| 残余风险 | 中 |
| 改进建议 | ① 小样本切片禁止输出（最低 N=10）；② 聚合查询最低人数限制；③ 报告中对学生 ID 脱敏 |

#### 风险 3：未经授权的数据访问

| 维度 | 内容 |
|------|------|
| 可能性 | 低 |
| 影响 | 高 |
| 风险等级 | 中 |
| 现有措施 | JWT 认证、RBAC、TOCTOU 防护、路径遍历防护 |
| 残余风险 | 低 |
| 改进建议 | ① 定期权限复核（季度）；② 异常访问实时告警；③ 敏感操作二次验证 |

#### 风险 4：数据保留超期

| 维度 | 内容 |
|------|------|
| 可能性 | 中 |
| 影响 | 中 |
| 风险等级 | 中 |
| 现有措施 | 分析事件 90 天自动清理 |
| 残余风险 | 中 |
| 改进建议 | ① 建立全量数据过期清理机制（含评估结果、预警记录、干预计划）；② 清理任务纳入定时调度与监控 |

### 1.4 数据主体权利保障

| 权利类型 | 实现方式 | 状态 |
|---------|---------|------|
| 知情权 | 注册时告知数据使用目的、范围、期限 | 待实现 |

| 删除权 | 用户可申请删除（法律要求保留的除外，如预警记录 5 年） | 待实现 |

## 二、危机升级 SOP

### 2.1 危机识别

**触发条件**（满足任一即触发危机升级流程）：

- 自评风险等级 ≥ 4（critical）
- 文本评估检测到自伤/自杀关键词
- 咨询师手动标记为"危机"

### 2.2 升级路径

```
Level 1: 系统自动预警
  ↓ (≤15 分钟)
Level 2: 咨询师确认/评估
  ↓ (≤2 小时)
Level 3: 紧急联系学生
  ↓ (如无法联系或情况紧急)
Level 4: 转介专业机构/校方紧急响应
```

### 2.3 各级别响应要求

| 级别 | 响应人 | 响应时限 | 行动 |
|------|--------|---------|------|
| L1 | 系统 | 即时 | 推送预警通知给绑定咨询师（站内信 + 邮件） |
| L2 | 咨询师 | ≤15 分钟 | 查看评估详情、初步评估风险严重性 |
| L3 | 咨询师 | ≤2 小时 | 联系学生（电话/面谈）、进行面谈评估 |
| L4 | 咨询师 + 校方 | 即时 | 转介专业精神卫生机构、启动校方紧急响应预案 |

**升级规则**：
- L2 超时未响应 → 自动升级至心理中心负责人
- L3 无法联系学生 → 直接升级至 L4
- L4 启动后须同步通知校方学工部门与保卫处

### 2.4 危机记录

- 所有危机事件须在系统中记录，不得事后补录或修改
- **记录内容**：触发时间、触发条件、响应人、各级响应时间戳、处置措施、结果、复盘结论
- **保留期限**：5 年（与预警记录一致）
- **上报要求**：重大危机事件（涉及自伤/自杀实施、需转介精神卫生机构）须在 24 小时内上报校方
- **审计要求**：危机记录纳入 OperationLog 审计范围，任何访问均留痕

### 2.5 危机后复盘

- **复盘时限**：每次危机事件后 48 小时内召开复盘会
- **参与人员**：当事咨询师、心理中心负责人、系统运营代表
- **评估维度**：
  - 识别是否及时（系统预警是否触发）
  - 响应是否适当（是否符合 SOP）
  - 系统是否有效（流程是否顺畅、工具是否可用）
- **输出**：改进措施清单，明确责任人与完成时限，跟踪闭环

## 三、人工复核 SLA

### 3.1 复核范围

- 所有高风险（level ≥ 3）评估结果
- 所有危机预警（level = 4）
- 模型结果与咨询师判断差异 ≥ 2 级的案例
- 随机抽样的低风险案例（每月不低于 5%），用于监控模型漂移

### 3.2 复核流程

1. 系统自动标记需复核的案例，推送至咨询师工作台
2. 咨询师在 SLA 时限内完成复核
3. 复核结果记录：**确认** / **修正** / **误报**
4. 修正后的结果反馈给模型训练流程（用于后续迭代）
5. 误报案例填写误报报告（见第四节）

### 3.3 SLA 标准

| 优先级 | 响应时限 | 复核时限 | 升级条件 |
|--------|---------|---------|---------|
| P0-危机 | ≤15 分钟 | ≤2 小时 | 超时自动上报校方 |
| P1-高风险 | ≤1 小时 | ≤24 小时 | 超时通知心理中心负责人 |
| P2-中风险 | ≤4 小时 | ≤72 小时 | 超时记录但不上报 |
| P3-差异复核 | ≤24 小时 | ≤1 周 | 超时纳入模型监控 |

**说明**：
- 响应时限：咨询师从收到通知到开始查看案例的时间
- 复核时限：咨询师从开始查看到提交复核结果的时间
- 超时升级：系统自动触发，无需人工干预

### 3.4 复核质量指标

| 指标 | 目标值 | 监控频率 |
|------|--------|---------|
| 复核一致率（咨询师与模型一致） | ≥ 85% | 月度 |
| 误报率 | ≤ 15% | 月度 |
| 漏报率 | ≤ 5% | 月度 |
| 平均复核时长 | 见 SLA 标准 | 月度 |
| SLA 达成率 | ≥ 95% | 月度 |


### 4.3 月度汇总报告

每月汇总当月误报/漏报情况，输出以下内容：

- **误报总数 / 误报率**
- **漏报总数 / 漏报率**
- **按风险等级分布**（低/中/高/危机）
- **按原因分类**（见 4.1、4.2 原因选项）
- **趋势分析**（与上月、近三月对比）
- **改进措施进展**（跟踪上月改进措施的落地情况）

## 五、试点协议要点

试点启动前，数据控制者（试点高校）与数据处理者（DWS 运营方）须签署书面协议，明确以下要点：

| 要点 | 内容要点 |
|------|---------|
| 数据控制者/处理者职责 | 明确双方角色与职责边界，控制者决定目的与方式，处理者按指令处理 |
| 使用目的限制 | 仅用于学生心理健康风险评估与干预，不得挪作他用 |
| 保存期限 | 按本文档 1.2 节执行，期满自动清理 |
| 访问审批流程 | 咨询师访问需绑定关系，管理员访问需审批留痕 |
| 事件响应机制 | 按本文档第二节危机升级 SOP 执行 |
| 退出机制 | 试点结束后数据返还或销毁，销毁须出具证明 |
| 联系人 | 双方各指定 1 名主联系人 + 1 名备用联系人，7×24 可达 |
| 审计权 | 控制者有权对处理者进行安全审计，提前 7 个工作日通知 |
| 违约责任 | 违反协议约定须承担相应责任，包括但不限于数据泄露赔偿 |
