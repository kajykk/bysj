# Phase 2 研究协议

> 本文档定义 DWS 心理健康风险评估系统 Phase 2 用户体验研究的执行协议，涵盖半结构化访谈（≥10 次）与可用性测试（≥8 人）的全流程规范。

---

## 一、研究目标

Phase 2 研究围绕以下核心目标展开，研究结果将直接服务于 Phase 3 受控试点的方案设计：

1. **验证 4 个 Persona 的行为假设**：通过真实用户访谈与行为观察，确认主动型、被动型、高风险型学生与咨询师四类 Persona 的核心行为模式是否与设计假设一致。
2. **识别核心旅程中的体验缺陷**：定位评估填写、结果展示、预警处置、干预跟进等关键触点上的可用性瓶颈与情感摩擦点。
3. **测量任务成功率和 SUS 分数**：量化 5 个核心场景的独立完成率与系统可用性量表（SUS）得分，建立 Phase 3 优化的基线指标。
4. **为 Phase 3 受控试点提供证据**：输出可操作的优化建议清单、风险等级与体验缺陷映射表，作为 Phase 3 试点范围与优先级排序的依据。

---

## 二、半结构化访谈大纲（≥10 次）

### 2.1 访谈对象招募

- **目标人数**：10-15 名参与者
- **构成**：
  - 6-8 名学生（覆盖主动型 / 被动型 / 高风险型三类 Persona）
  - 3-4 名咨询师（含一线辅导员与专职心理教师）
  - 1-2 名管理员（心理中心或学生工作负责人）
- **招募渠道**：
  - 校方心理健康教育与咨询中心官方推荐
  - 辅导员定向推荐（覆盖不同年级与专业）
  - 校内公告栏、班级群自愿报名
- **筛选标准**：
  - 年满 18 周岁
  - 在校学生或在职教职工
  - 签署书面知情同意书
  - 排除标准：当前处于急性心理危机状态（需直接转介危机干预通道，不纳入本研究）

### 2.2 访谈问题框架

#### 学生组（60 分钟）

1. **开场（5 分钟）**：自我介绍、研究目的、隐私说明、录音许可确认
2. **背景了解（10 分钟）**：
   - 专业、年级、日常心理健康关注情况
   - 是否使用过心理评估工具（线上或线下）
   - 对心理健康评估的总体态度
3. **系统使用体验（20 分钟）**：
   - 首次使用感受（注册、登录、首屏引导）
   - 评估流程的难易程度（题量、节奏、术语理解）
   - 结果展示是否清晰（风险等级、雷达图、趋势曲线）
   - 隐私担忧（数据存储、咨询师可见范围、家长知情）
4. **预警与干预（15 分钟）**：
   - 是否收到过预警通知（短信 / 应用内 / 邮件）
   - 预警内容是否引起焦虑或抵触
   - 干预建议是否可执行（自助资源、预约通道、紧急联系）
5. **改进建议（10 分钟）**：
   - 最希望改进的功能（排序 Top 3）
   - 最不喜欢的体验（具体场景描述）

#### 咨询师组（60 分钟）

1. **开场（5 分钟）**
2. **工作背景（10 分钟）**：
   - 负责学生数量与构成
   - 日常风险筛查流程（纸质问卷 / 口头访谈 / 线上工具）
   - 现有工具的主要痛点
3. **系统使用（20 分钟）**：
   - 预警列表的可用性（排序、筛选、状态标记）
   - 风险详情是否足够（特征贡献、历史趋势、上下文备注）
   - 模型结果的可信度（是否愿意据此做出干预决策）
4. **干预工作流（15 分钟）**：
   - 干预计划记录是否便捷（模板、字段、附件）
   - 与学生沟通渠道（系统内消息 / 外部渠道）
   - 复评跟进机制（提醒、闭环、归档）
5. **改进建议（10 分钟）**

### 2.3 访谈记录与分析

- **记录方式**：录音（需征得参与者书面同意）+ 现场关键笔记 + 观察员补充记录
- **分析方法**：主题分析法（Thematic Analysis）
  1. 编码：逐句开放编码，形成初始代码本
  2. 聚类：将相似代码归并为候选主题
  3. 主题：提炼核心主题并建立主题间关系图
- **输出**：访谈发现报告（含主题清单、引用证据、Persona 验证结论）

## 三、可用性测试方案（≥8 人）

### 3.1 测试设计

- **参与者**：8-12 名（至少 5 名学生、3 名咨询师）
- **环境**：安静会议室，使用测试设备（iPhone / Android / PC 三端各覆盖）
- **方式**：Think Aloud（出声思考法），观察员同步记录行为与言语
- **时长**：每位参与者 45-60 分钟
- **录制**：屏幕录制 + 摄像（需签署知情同意书）

### 3.2 测试任务（5 个核心场景）

#### 任务 1：首次完成风险评估（学生）

- **前置条件**：已注册并登录系统
- **步骤**：找到评估入口 → 完成结构化评估（≥20 题）→ 查看结果页
- **成功标准**：独立完成全流程，无向观察员求助
- **测量指标**：完成时间、错误次数、求助次数、中途退出率

#### 任务 2：查看风险趋势与干预建议（学生）

- **前置条件**：已有 ≥3 次评估记录
- **步骤**：查看趋势图 → 理解风险等级 → 查看干预建议
- **成功标准**：能用自己的语言解释当前风险等级与近期趋势变化
- **关注点**：趋势图标注是否清晰、干预建议是否与风险等级匹配

#### 任务 3：处理一条预警（咨询师）

- **前置条件**：预警列表中有 ≥1 条待处理预警
- **步骤**：查看预警 → 评估风险详情 → 记录处置意见 → 创建干预计划
- **成功标准**：完成全流程，处置记录字段完整、干预计划可执行
- **关注点**：处置状态流转是否符合 SOP、干预计划模板是否覆盖必要字段

#### 任务 4：导出报告（学生 / 咨询师）

- **步骤**：找到导出入口 → 选择格式（PDF / Excel）→ 生成并下载
- **成功标准**：成功下载文件并能在本地打开
- **关注点**：异步任务通知是否可见、文件命名是否规范、生成耗时是否在可接受范围

#### 任务 5：使用帮助系统（学生）

- **步骤**：找到帮助入口 → 查看 FAQ → 使用反馈表单提交一条反馈
- **成功标准**：能找到所需信息并完成反馈提交
- **关注点**：帮助入口可达性、FAQ 分类是否符合心智模型、反馈表单字段合理性

### 3.3 测量指标

- **任务成功率**：目标 ≥90%（5 个任务各自独立统计）
- **完成时间**：每个任务的耗时分布（中位数与 P90）
- **错误次数**：偏离主路径的操作次数
- **求助次数**：向观察员求助的次数
- **SUS 量表（系统可用性量表）**：目标 ≥75 分
- **单一可用性测量（UME，Single Ease Question）**：每个任务结束后立即采集

### 3.4 SUS 量表（标准 10 题）

> 计分规则：1-5 分李克特量表；正向题（第 1、3、5、7、9 题）正常计分，反向题（第 2、4、6、8、10 题）反转计分；总分 = 各项转换分之和 × 2.5，满分 100 分。

1. 我认为我会频繁使用这个系统
2. 我发现系统 unnecessarily complex
3. 我认为系统容易使用
4. 我认为需要技术支持才能使用
5. 各功能整合良好
6. 系统中存在不一致
7. 其他人会很快学会使用
8. 系统使用起来很 awkward
9. 使用时有信心
10. 需要先学习很多内容

## 五、研究伦理

- **知情同意**：所有参与者须签署知情同意书，签署前充分告知研究目的、流程、风险与权利
- **心理安全保障**：涉及心理健康话题时，现场配备紧急联系信息（校心理中心 24 小时热线、危机干预值班电话）
- **危机升级 SOP**：发现高风险信号（如自伤意念、自杀计划）时，按危机升级 SOP 立即启动响应流程，优先保障参与者安全，研究活动让位于干预
- **数据最小化原则**：仅记录研究必要信息，不采集与本研究无关的敏感数据（如身份证号、具体诊断记录）
- **伦理审批**：研究方案须报校方伦理委员会审批，未经批准不得启动研究

## 六、执行时间表

- **第 3 月**：访谈大纲定稿 + 招募参与者 + 伦理审批 + 测试环境与脚本准备
- **第 4 月**：执行访谈（≥10 次）+ 可用性测试（≥8 人）+ 同步整理原始数据
- **第 5 月**：数据分析 + 发现报告 + 优化建议清单 + Phase 3 试点方案输入
