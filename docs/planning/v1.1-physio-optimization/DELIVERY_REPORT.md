## 1. 项目概述

### 1.1 背景
当前抑郁症预警系统的生理模型使用启发式规则，性能严重不足：
- F1-Score: 0.6667
- Accuracy: 0.5000（所有低风险样本被误分类为高风险）
- Recall: 1.0000（过度预测）

### 1.2 目标
通过引入深度学习模型（PyTorch MLP），将生理模型升级为数据驱动的预测引擎：
- **F1-Score ≥ 0.80**（提升 20%）
- **Accuracy ≥ 0.75**（提升 50%）
- **Precision ≥ 0.70**（提升 40%）
- **Recall ≥ 0.80**（更平衡）
- **ROC-AUC ≥ 0.85**（新增）

### 2.2 核心设计决策

#### 模型架构（轻量级 MLP）
```
输入层 (13 features)
    ↓
BatchNorm1d
    ↓
Dense(64) + ReLU + BatchNorm1d
    ↓
Dropout(0.4)
    ↓
Dense(32) + ReLU
    ↓
Dropout(0.3)
    ↓
Dense(16) + ReLU
    ↓
Dense(1) + Sigmoid
```
- **参数量**: ~3,500（适合 ~2,000 样本的小数据集）
- **推理延迟**: < 50ms（CPU）

#### 数据流程（严格防泄漏）
```
原始数据
    ↓
特征工程（13 维特征）
    ↓
分层划分 (70% / 15% / 15%)
    ├── 训练集 ← 唯一可用于学习的集合
    ├── 验证集 ← 仅用于超参数选择和早停
    └── 测试集 ← 全程锁定，仅用于最终报告
    ↓
仅对训练集应用 SMOTE（采样比例 0.5~0.8）
    ↓
StandardScaler.fit() 仅在训练集执行
    ↓
模型训练 + 交叉验证
    ↓
测试集最终评估
```

## 3. 科学严谨性保障

### 3.1 关键问题识别与修复

| # | 问题 | 严重程度 | 修复措施 |
|---|------|----------|----------|
| 1 | SMOTE 数据泄漏风险 | 🔴 严重 | 明确流程：划分 → 仅训练集 SMOTE |
| 2 | 模型过于复杂 | 🔴 严重 | 缩减为轻量级 MLP，参数 < 5,000 |
| 3 | 交叉验证流程不严谨 | 🟡 中等 | 每折内部独立预处理 |
| 4 | 缺乏统计显著性检验 | 🟡 中等 | 添加 McNemar 检验 + Bonferroni 校正 |
| 5 | 超参数搜索范围过大 | 🟢 轻微 | 减少 trials (100→50) |

### 3.2 应用的最佳实践

1. **数据泄漏防护**
   - SMOTE 仅在训练集上应用
   - StandardScaler.fit() 仅在训练集执行
   - 验证集和测试集保持原始分布

2. **小数据集深度学习策略**
   - 轻量级架构（~3,500 参数）
   - 强正则化（Dropout 0.4→0.3→0.2，weight_decay 1e-4~1e-2）
   - 早停 + 学习率衰减 + 梯度裁剪

3. **严格评估协议**
   - 5 折交叉验证（每折独立预处理）
   - Hold-out 测试集（全程不参与训练决策）
   - Bootstrap 95% CI
   - McNemar 统计检验（vs 启发式模型）
   - Bonferroni 多重比较校正

## 6. 风险评估与缓解

| 风险 | 可能性 | 影响 | 缓解措施 |
|------|--------|------|----------|
| 数据分布不匹配 | 高 | 高 | 仔细预处理；领域适应 |
| 过拟合（小数据集） | 中 | 高 | 轻量级模型 + 强正则化 + 早停 |
| PyTorch 依赖问题 | 低 | 中 | 自动回退启发式规则 |
| 推理延迟增加 | 低 | 低 | torch.no_grad()；ONNX 加速 |
| 类别不平衡 | 高 | 中 | Focal Loss + SMOTE + 类别权重 |
