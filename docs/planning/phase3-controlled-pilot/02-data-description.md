# DWS 数据说明（Data Description）

> 本文档描述 DWS 心理健康风险评估系统在受控试点期间采集、处理和存储的数据，遵循"最小化采集、去标识化存储、有限目的使用"原则。文档应随数据流变化而同步更新。

---

## 一、数据分类分级

### 1.1 数据分级

| 等级 | 定义 | 示例 | 保护措施 |
|------|------|------|---------|
| L1 - 公开 | 可公开的数据 | 系统公告、帮助文档 | 无特殊保护 |
| L2 - 内部 | 内部运营数据 | 聚合统计、系统日志 | 访问控制 |
| L3 - 个人 | 可识别个人身份 | 用户 ID、年级、性别 | 去标识化 + 访问控制 |
| L4 - 敏感个人 | 敏感健康信息 | 风险评估结果、问卷得分 | 加密 + 去标识化 + 严格访问控制 |
| L5 - 高敏感 | 危机事件详情 | 自伤/自杀意图文本 | 强加密 + 最小知情范围 + 审计 |

### 1.2 数据类型清单

| # | 数据类型 | 分级 | 采集目的 | 保留期限 | 来源 |
|---|---------|------|---------|---------|------|
| D1 | 用户基本信息 | L3 | 身份识别、角色分配 | 账户注销后 90 天 | 注册 |
| D2 | 风险评估输入 | L4 | 模型预测输入 | 90 天 | 用户提交 |
| D3 | 模型预测结果 | L4 | 风险预警、复核 | 90 天 | 模型输出 |
| D4 | 复核记录 | L4 | 人工复核、质量追踪 | 180 天 | 咨询师 |
| D5 | 危机事件 | L5 | 危机干预、合规审计 | 365 天 | 系统检测 |
| D6 | 行为埋点 | L3 | 产品优化、可用性分析 | 90 天 | 自动采集 |
| D7 | 审计日志 | L2 | 操作追溯、合规 | 365 天 | 系统记录 |
| D8 | 生理指标 | L4 | 生理模型输入 | 90 天 | 可穿戴设备 |

## 二、数据采集

### 2.1 采集原则

1. **最小化**：仅采集实现功能所需的最少数据
2. **知情同意**：所有数据采集需用户明确同意
3. **用途限定**：数据仅用于声明的目的，不挪作他用
4. **去标识化**：存储时去除直接标识符（姓名、学号等）
5. **可审计**：所有数据访问操作记录到审计日志

### 2.2 埋点事件（D6）

埋点事件遵循白名单策略，不采集问卷正文和敏感健康原文：

| 事件类型 | 采集字段 | 用途 | 敏感度 |
|---------|---------|------|--------|
| assessment_enter | page, timestamp | 进入评估页 | L3 |
| assessment_start | assessment_type | 开始评估 | L3 |
| assessment_complete | duration, risk_level | 完成评估 | L4 |
| assessment_abandon | abandon_point | 放弃评估 | L3 |
| warning_handle | action, response_time | 预警处理 | L4 |
| help_use | help_type | 使用帮助 | L3 |
| task_fail | task_type, error_code | 任务失败 | L2 |

**隐私保护措施**：
- 仅采集白名单字段（7 种事件类型 + 7 种元数据键）
- 元数据仅限标量值（字符串/数字/布尔），不采集对象
- 单字段长度限制 200 字符
- 保留期限 90 天
- 需用户同意（`analytics_consent`）
- 所有访问记录审计日志

### 2.3 评估输入数据（D2）

| 模态 | 输入字段 | 数据类型 | 去标识化方式 |
|------|---------|---------|-------------|
| 结构化 | PHQ-9/GAD-7 等量表得分 | 数值 | 不含姓名，仅关联 user_id |
| 文本 | 开放式回答文本 | 文本 | 存储前截断 200 字符，不采集原文 |
| 生理 | HRV、睡眠时长等 | 数值 | 设备 ID 去标识化 |

### 2.4 生理指标数据（D8）

| 指标 | 来源 | 采集频率 | 处理方式 |
|------|------|---------|---------|
| 心率变异性 (HRV) | 可穿戴设备 | 每日 | 去标识化存储 |
| 睡眠时长 | 可穿戴设备 | 每日 | 去标识化存储 |
| 步数 | 可穿戴设备 | 每日 | 去标识化存储 |
| 静息心率 | 可穿戴设备 | 每日 | 去标识化存储 |

> 注：生理数据采集需用户额外同意，且仅在用户主动上传时采集。

## 三、数据处理

### 3.1 去标识化流程

```
[原始数据输入]
       |
       v
[直接标识符去除] --> 姓名、学号、手机号 → 移除
       |
       v
[间接标识符处理] --> 出生日期 → 年龄段；邮编 → 前 3 位
       |
       v
[敏感内容脱敏] --> 文本输入 → 截断 200 字符 + 关键词检测
       |
       v
[关联标识符生成] --> user_id（自增整数，不可逆映射）
       |
       v
[加密存储] --> L4/L5 数据使用 PII 加密（app/core/pii_crypto.py）
```

### 3.2 PII 加密

- **算法**：AES-256-GCM（`app/core/pii_crypto.py`）
- **密钥管理**：环境变量 `PII_ENCRYPTION_KEY`，支持轮换
- **加密字段**：评估输入文本、危机事件详情、复核备注
- **审计**：所有解密操作记录到审计日志

### 3.3 数据清洗

生理模型数据清洗流程（[data_cleaner.py](../../backend/app/ml/data_cleaner.py)）：

1. 缺失值处理：连续变量中位数填充，分类变量众数填充
2. 异常值检测：IQR 方法识别并截断
3. 标准化：Z-score 标准化（[scaler.py](../../backend/app/ml/scaler.py)）
4. 特征工程：衍生特征生成（[feature_engineering.py](../../backend/app/ml/feature_engineering.py)）

## 四、数据存储

### 4.1 存储位置

| 数据类型 | 存储位置 | 加密 | 备份 |
|---------|---------|------|------|
| 用户信息 | PostgreSQL `users` 表 | 密码哈希 | 每日 |
| 评估数据 | PostgreSQL `assessments` 表 | PII 加密 | 每日 |
| 预测结果 | PostgreSQL `risk_records` 表 | - | 每日 |
| 复核记录 | PostgreSQL `review_tasks` 表 | PII 加密 | 每日 |
| 危机事件 | PostgreSQL `crisis_events` 表 | PII 加密 | 每日 |
| 埋点事件 | PostgreSQL `analytics_events` 表 | - | 每日 |
| 审计日志 | PostgreSQL `operation_logs` 表 | - | 每日 |
| 缓存 | Redis | - | 无 |
| 模型文件 | 文件系统 `models/` | - | 每周 |

### 4.2 访问控制

| 角色 | 可访问数据 | 访问方式 |
|------|-----------|---------|
| 普通用户 | 仅自己的数据 | API 鉴权 |
| 咨询师 | 所负责学生的评估/复核数据 | API 鉴权 + 角色权限 |
| 管理员 | 全部数据（需审计） | API 鉴权 + 管理员权限 |
| 系统 | 自动处理所需数据 | 服务层内部调用 |

### 4.3 数据保留与删除

| 数据类型 | 保留期限 | 删除方式 | 触发条件 |
|---------|---------|---------|---------|
| 用户账户 | 注销后 90 天 | 硬删除 | 用户主动注销 |
| 评估数据 | 90 天 | 硬删除 | TTL 自动清理 |
| 埋点事件 | 90 天 | 硬删除 | TTL 自动清理 |
| 审计日志 | 365 天 | 归档后删除 | TTL 自动清理 |
| 危机事件 | 365 天 | 归档后删除 | TTL 自动清理 |

**GDPR 权利实现**：
- 数据导出：`GET /api/v1/gdpr/export`（用户可导出自己的全部数据）
- 数据删除：`DELETE /api/v1/gdpr/delete`（用户可请求删除自己的数据）
- 数据访问：`GET /api/v1/user-data`（用户可查询自己的数据）

## 五、数据共享

### 5.1 内部共享

| 共享场景 | 数据范围 | 接收方 | 保护措施 |
|---------|---------|--------|---------|
| 风险预警 | 风险等级、预警信息 | 咨询师 | 角色权限控制 |
| 复核任务 | 评估摘要、风险指标 | 咨询师 | 角色权限控制 |
| 危机升级 | 危机事件详情 | 心理中心 | 最小知情范围 |

### 5.2 外部共享

- **原则**：不与第三方共享个人数据，除非获得用户明确同意
- **例外**：法律要求或危机事件需紧急联系监护人/医疗机构
- **聚合数据**：去除个人标识的聚合统计可用于研究发表

### 5.3 数据控制者/处理者

试点期间的角色定义（需试点协议明确）：

| 角色 | 主体 | 职责 |
|------|------|------|
| 数据控制者 | 试点高校 | 决定数据处理目的和方式 |
| 数据处理者 | DWS 开发团队 | 按控制者指示处理数据 |
| 数据保护官 | 校方指定 | 监督合规、处理用户权利请求 |

## 六、数据安全

### 6.1 技术措施

| 措施 | 实现 | 说明 |
|------|------|------|
| 传输加密 | HTTPS/TLS 1.2+ | 所有 API 通信 |
| 存储加密 | AES-256-GCM | PII 字段加密 |
| 密码哈希 | bcrypt | 用户密码 |
| 访问控制 | RBAC + 权限 | 角色权限模型 |
| 审计日志 | OperationLog | 所有敏感操作 |
| 速率限制 | 速率限制器 | 防暴力破解 |
| 输入验证 | Pydantic | 防 SQL 注入/XSS |

### 6.2 安全事件响应

| 事件等级 | 定义 | 响应时限 | 措施 |
|---------|------|---------|------|
| P0 - 严重 | 数据泄露/未授权访问 | 1 小时 | 暂停服务 + 应急响应 |
| P1 - 高危 | 权限滥用/异常访问 | 4 小时 | 调查 + 限制访问 |
| P2 - 中危 | 配置错误/日志异常 | 24 小时 | 修复 + 监控 |
| P3 - 低危 | 潜在风险 | 72 小时 | 评估 + 记录 |

## 七、数据质量

### 7.1 质量指标

| 指标 | 目标值 | 监控方式 | 频率 |
|------|--------|---------|------|
| 数据完整性 | ≥ 99% | 必填字段缺失率 | 每日 |
| 数据准确性 | ≥ 95% | 抽样校验 | 每周 |
| 数据及时性 | ≤ 1s 延迟 | API 响应时间 | 实时 |
| 数据一致性 | ≥ 98% | 跨表校验 | 每周 |

### 7.2 数据漂移检测

- **工具**：[drift_detector.py](../../backend/app/ml/drift_detector.py)
- **方法**：KL 散度、PSI（群体稳定性指数）
- **频率**：每日
- **告警阈值**：KL 散度 > 0.3 或 PSI > 0.25
- **响应**：告警 → 评估影响 → 决定是否暂停/重训练

## 八、试点期间数据采集计划

### 8.1 行为指标采集

| 指标 | 采集方式 | 频率 | 目标值 |
|------|---------|------|--------|
| 评估完成率 | 埋点 assessment_complete / assessment_start | 每日 | ≥ 70% |
| 放弃点分布 | 埋点 assessment_abandon | 每日 | - |
| 异步任务成功率 | 任务日志 | 每日 | ≥ 95% |
| 预警响应时长 | 复核任务时间戳 | 每日 | 中位 ≤ 4h |
| 复核一致率 | 复核结果对比 | 每周 | ≥ 85% |
| 帮助使用率 | 埋点 help_use | 每日 | - |
| 满意度与 NPS | 问卷 | 每月 | NPS ≥ 30 |

### 8.2 模型验证数据采集

- **采集方式**：以专业量表/专家复核作为参照
- **采集频率**：每两周一次
- **验证内容**：sensitivity、specificity、PPV、NPV、AUROC、校准度
- **公平性检查**：按性别、年级分组（`min_group_size=30` 保护）
- **工具**：`POST /api/v1/validation/clinical`
