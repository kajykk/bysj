# WF-1 快速止血 — 前端首屏性能优化（ISS-03）

> 阶段：WF-1 | 技能：sys-perf-diagnosis | 日期：2026-07-14
> 原则：先数据后决策、低成本高收益、伴随验证与可回滚。

## 1. 诊断依据（来自既有 Lighthouse 产物，非猜测）

`frontend/lighthouse-prod.json` 实测：

| 指标 | 值 | 判读 |
|---|---|---|
| performance | 53/100 | 需改进 |
| FCP | 5.8s | 严重（render-blocking）|
| LCP | 6.8s | 严重 |
| TTI | 7.2s | 严重 |
| total-byte-weight | 538 KiB | **不大** → 非体积问题 |
| unused-javascript | 250 KiB | 首屏加载了非必要 JS |

**关键判断**：传输体积仅 538 KiB 但 FCP/LCP 高达 5.8/6.8s，说明瓶颈在**关键渲染路径的阻塞**（同步 import + 首屏执行），而非下载体积。定位到 `main.ts`：
- `element-plus/theme-chalk/dark/css-vars.css` 同步 import 进主 chunk（浅色首屏无需暗色变量）。
- Sentry 在 mount 前同步初始化，占用首屏主线程。

## 2. 实施改动（最小化、可回滚）

| 文件 | 改动 | 收益 |
|---|---|---|
| `frontend/src/composables/useTheme.ts` | 暗色 CSS 改为按需 `import()`，仅当 `isDark` 为真时加载 | 浅色首屏移除暗色 CSS 解析，缩短关键 CSS 路径 |
| `frontend/src/main.ts` | Sentry 改为 mount 后异步 `import('./plugins/sentry')`；首屏用空实现缓冲 `captureException` | Sentry 拆为独立 async chunk（`sentry-*.js`），移出首屏主线程执行 |

改动规模：2 文件，+18/-3 行（`git diff --stat` 实测）。

## 3. 构建验证（实测）

- 构建产物确认新增独立 `dist/assets/sentry-*.js` chunk → Sentry 已脱离首屏执行路径。
- `manualChunks` 已有良好拆分（vue-core/router/i18n/echarts/element-plus 分组），路由已全部懒加载（`() => import()`）。
