## 📊 状态流转图 (State Flow)

```text
新建 ──→ 已确认 ──→ 修复中 ──→ 待复核 ──→ 已关闭
                ↓                       ↓
              暂缓/拒绝              重新打开
```

### 状态说明
- **新建**: 审查发现但未确认。
- **已确认**: 负责人确认是有效问题。
- **修复中**: 已分配责任人，正在修复。
- **待复核**: 修复完成，等待验证。
- **已关闭**: 复核通过。
- **暂缓**: 非本轮必须修复，需写明理由与延期到何时。
- **拒绝**: 不是问题，需写明理由。

## 🛡️ 修复跟踪规则（计划五.4）

- P0/P1 每日同步状态。
- 每个问题必须有关联提交或明确说明无需代码修改。
- 不允许"口头关闭"。
- 修复后必须补充或更新测试（写入 `06-regression-tests.md`）。
- 同类问题必须横向排查（Iron Rule #9）。
- 涉及权限、安全、数据一致性的问题必须由第二人复核（Iron Rule #10）。
- 修复引入新问题时，重新打开原问题或创建关联问题。

## 📝 使用示例 (Usage Examples)

### 记录新问题
```text
> 用户：发现咨询师列表 API 越权问题，P1

操作：
1. 在问题记录表追加一行：ISS-002 | P1 | counselor API | 后端 | Counselor list API leaks users outside authorization scope | 2026-06-29 | auditor | 新建 | - | - | -
2. 在问题详情区追加 ISS-002 完整模板。
3. 同步更新统计汇总与 AUDIT_STATE.md。
```

### 提交修复
```text
> 用户：ISS-002 修复完成，commit abc1234

操作：
1. 在问题记录表中将 ISS-002 状态改为"待复核"。
2. 在问题详情区填写：修复方案 / 关联提交 abc1234 / 回归测试步骤。
3. 在 06-regression-tests.md 创建对应回归用例 REG-ISS-002。
4. 同步更新统计汇总与 AUDIT_STATE.md。
```
