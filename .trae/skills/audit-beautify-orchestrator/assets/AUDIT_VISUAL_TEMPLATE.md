## ⚡ 性能优化检查 (Performance Optimization)

> 对应计划八。

### 构建与资源
- [ ] 保持路由懒加载
- [ ] 检查 Vite `manualChunks` 分包策略
- [ ] ECharts、Element Plus、Sentry 等较大依赖避免进入首屏主包
- [ ] 开启 gzip/brotli
- [ ] 静态资源设置合理缓存
- [ ] 删除未使用依赖和未使用组件
- [ ] 图片压缩和懒加载
- [ ] SVG 图标按需导入

### 运行时
- [ ] 图表组件销毁时释放实例
- [ ] 统一 resize 监听，避免每个图表单独监听
- [ ] 长列表分页或虚拟滚动
- [ ] 避免 watch 过深对象
- [ ] 避免在模板中执行复杂计算
- [ ] 搜索框使用 debounce
- [ ] 请求去重或缓存低频变化数据
- [ ] 页面切换时取消无效请求

### 网络请求
- [ ] 合并首屏必要接口
- [ ] 非首屏数据延迟加载
- [ ] Dashboard 数据可按卡片独立加载
- [ ] 接口失败支持重试
- [ ] 导出/PDF/训练等长任务改为任务式轮询或 WebSocket 推送

### 后端配合
- [ ] 高频接口增加索引
- [ ] Dashboard 统计接口可以做缓存
- [ ] 大型导出使用异步任务
- [ ] PDF 生成放入后台任务或线程池
- [ ] 模型推理预加载
- [ ] 数据库查询使用 selectinload/joinedload 避免 N+1
- [ ] API 返回字段按页面需要裁剪
- [ ] 日志和指标避免记录超大 payload
- [ ] Redis 连接池复用
- [ ] 限制分页最大 `page_size`
